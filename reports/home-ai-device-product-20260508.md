## 1. 硬件载体选型对比

### 方案 A：带屏智能音箱（小度/小爱/天猫精灵）

| 维度 | 说明 |
|------|------|
| **价格** | 600–2000 元（旗舰款如小度智能屏 X10 约 1500 元） |
| **开发难度** | **低** — SDK 开放，但定制化空间有限，技能生态封闭 |
| **视频能力** | 一般（最高 1080p 摄像头，但只用于视频通话，不支持人脸识别本地处理） |
| **语音能力** | 成熟唤醒+连续对话，但语音模型受设备算力限制，依赖云端 |
| **外观** | 消费级家电设计，体面但无法定制 |
| **维护成本** | 低（官方 OTA 更新），但功能完全受制于厂商 |
| **适合场景** | 快速验证概念，不适合深度定制 |

**结论：** 适合作为入门方案，但「家庭管家」角色会受限于厂商生态。

### 方案 B：iPad + 支架方案

| 维度 | 说明 |
|------|------|
| **价格** | iPad 第九代（约 2500 元）+ 充电底座/壁挂支架（约 300 元）= 约 2800 元 |
| **开发难度** | **低–中** — iOS 开发框架成熟，可完全控制 App 层；本地 AI 可通过 Core ML 运行 |
| **视频能力** | 优秀（1200 万摄像头，1080p 视频通话，支持人脸识别） |
| **语音能力** | iOS 语音助手成熟，Siri 生态完整；可接 ChatGPT API 做对话层 |
| **外观** | 取决于支架，整体感强，Apple 设计语言加分 |
| **维护成本** | 低（App 更新即可），但设备生命周期受 Apple 系统更新影响 |
| **致命缺点** | iPad 电池/屏幕始终开启有损耗风险；不适合「永远在线」的固定场景 |

**结论：** 性价比最高的方案，Apple 生态完善，适合快速落地。

### 方案 C：Raspberry Pi 4 + 摄像头模块 + 麦克风阵列 + 屏幕

| 维度 | 说明 |
|------|------|
| **价格** | 约 800–1200 元（RPi 4 约 400 元，摄像头 150 元，ReSpeaker 4-Mic 约 120 元，7" 触控屏 300 元） |
| **开发难度** | **高** — 需要自行搭 Linux 环境、音频驱动、视频流服务、AI 模型本地部署 |
| **视频能力** | 中等（CSI 摄像头最高 12MP，支持人脸识别但算力有限，需要优化） |
| **语音能力** | 取决于麦克风方案；ReSpeaker 阵列支持语音唤醒，但持续对话需要额外开发 |
| **外观** | 明显的「DIY 感」，外壳难处理，散热需要设计 |
| **维护成本** | 高（需要自己维护 OS、依赖更新），可靠性不如商用设备 |
| **适合场景** | 技术团队有嵌入式经验，想要完全控制硬件和软件 |

**结论：** 成本最低但工程量最大，适合技术团队做深度定制。

## 2. 交互设计

### 2.1 唤醒与对话模式

**方案一：唤醒词 + 持续对话（推荐）**
- 唤醒词：「Hey Thomas」「小管家」等自定义唤醒词
- 唤醒后进入持续对话状态（类似 Siri/Alexa），无需每次都喊唤醒词
- 优点：体验流畅，符合「家庭管家」定位
- 实现方案：Picovoice Porcupine（本地唤醒词，免费）或 Snowboy

**方案二：按钮触发**
- 物理按键/屏幕触控触发对话
- 优点：避免误唤醒，隐私性强
- 缺点：交互不够自然，破坏「管家」人设

**推荐：唤醒词方案**，结合「请按按钮确认敏感操作」的双重保障

### 2.2 屏幕内容设计

核心原则：**主屏展示状态，副屏承载交互**

| 显示内容 | 优先级 | 说明 |
|----------|--------|------|
| 情感表情/头像 | **高** | 家庭管家有「人格」，屏幕显示当前情绪状态（如 😊 开心、🤔 思考中） |
| 时钟/日期 | **高** | 基础信息，永远显示 |
| 天气 | 中 | 附带显示，不需要主动播报 |
| 家务提醒 | 中 | 今日任务列表，做完自动消项 |
| 音乐/媒体状态 | 中 | 正在播放时显示进度条 |
| 未读消息摘要 | 低 | 有飞书/微信消息时显示红点 |

**产品参考：** Amazon Echo Show 的显示逻辑值得借鉴——信息分层，常用信息常驻，功能性信息按需弹出。

### 2.3 多人场景处理

家庭场景比单人办公场景复杂很多：

1. **声纹识别** — 区分家庭成员，个性化响应（如「爸爸的日程」「妈妈的提醒」）
   - 实现：Pyaudio + 声纹模型（如 Resemblyzer），本地运行
2. **多身份优先级** — 主人（声纹验证） > 其他成员
   - 敏感操作（发消息、购物）需要主人确认
3. **「当前用户」感知** — 通过摄像头检测谁在设备前，自动切换身份
   - 实现：人脸识别（Face Recognition + OpenCV）
4. **隐私边界** — 儿童模式下限制某些功能；主人不在时自动进入「访客模式」

## 5. Thomas 建议

### 推荐方案：**iPad + 支架**（方案 B）

**理由：**
1. **最快落地** — App 开发周期短，Apple 生态完善
2. **成本可控** — 一次性硬件成本约 3000 元
3. **体验最好** — 屏幕、摄像头、麦克风都是消费级最优解
4. **可扩展** — iOS App 可以对接 Boss 的 Mac Mini 上的本地 LLM（通过 HomeKit 或自定义 API）
5. **维护成本低** — App 远程更新，无需物理接触设备
