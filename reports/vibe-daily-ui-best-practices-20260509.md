## 一、卡片 UI 设计

### 1.1 竞品参考分析

#### 竞品一：微信看一看 / 腾讯新闻小程序

**截图描述（典型新闻卡片）：**
- 卡片左侧配图（16:9 比例缩略图，约 80×80px）
- 右侧信息区：标题（2 行 max，17pt 粗体）、来源标签 + 时间（12pt 灰色）、摘要（2 行，14pt）
- 底部：互动栏（评论数、点赞数）灰底细线分隔

**信息层级：**
```
[图片] | 标题（粗体）           ← 第一眼
       | 来源 · 时间            ← 第二眼（灰色小字）
       | 摘要文字              ← 第三眼
       | ────────────────────
       | ❤️ 12  💬 8           ← 底部互动
```

**卡片间距：** 上下间距 8-12rpx，左右 padding 16rpx

### 1.2 视觉规范建议（基于微信官方设计指南）

**微信设计指南核心原则：**
1. **友好礼貌** — 重点突出，减少干扰元素
2. **清晰明确** — 导航明确，来去自如
3. **统一稳定** — 页面间一致性

**字号规范（iPhone6 基准 375px）：**

| 场景 | 字号 | 颜色 | 备注 |
|------|------|------|------|
| 卡片标题 | 17pt（34rpx） | #000000 | 最多 2 行，溢出省略 |
| 摘要正文 | 14pt（28rpx） | #666666 | 最多 2 行 |
| 来源/时间 | 12pt（24rpx） | #999999 | 横向排列 |
| 毒舌点评 | 13pt（26rpx） | #FF6B6B 或品牌色 | 斜体或特殊样式突出 |
| 分类标签 | 10pt（20rpx） | 品牌主色 | 圆角小标签 |

**间距规范：**

| 元素 | 间距 |
|------|------|
| 卡片左右 padding | 16rpx |
| 卡片上下间距 | 12rpx |
| 标题与摘要间距 | 8rpx |
| 卡片与底部互动栏 | 10rpx |

**色彩规范（建议品牌色系）：**

- 品牌主色：#FF6B6B（活力红，符合"毒"调性）
- 强调色：#4ECDC4（薄荷绿，用于关键操作）
- 背景色：#F7F7F7（浅灰，卡片白色 #FFFFFF）
- 文字主色：#333333
- 文字辅助色：#999999

```

│ ┌────┐                                      │

## 二、详情页交互

### 2.1 竞品参考

#### 微信读书 / 知乎日报详情页

**交互流程：**

```
[列表页] → 点击卡片 → [右滑过渡动画 300ms ease-out] → [详情页]
                                                         ↓
                                                    [顶部毛玻璃导航栏]
                                                    [大图 Banner]
                                                    [标题 + 来源]
                                                    [正文内容]
                                                    [毒舌点评高亮块]
                                                    [原文跳转按钮]
                                                    [底部分享栏]
```

**过渡动画规范（微信官方建议）：**
- 页面切换：默认无动画（微信统一处理）
- 自定义动画：建议 300ms 以内，ease-out 曲线
- 卡片展开：scale(0.95) → scale(1) + opacity 渐显
- 不要使用：超过 400ms 的动画、ease-in-out 以外的复杂曲线

**详情页排版规范：**

1. **顶部 Banner**：宽度 100%，高度 400rpx，cover 填充，底部渐变遮罩叠加标题
2. **内容区**：白色卡片，左上角大标题（20pt 粗体），正文 15pt 常规
3. **毒舌点评块**：灰色浅底背景（#F5F5F5），左侧 4rpx 品牌色竖线，斜体文字
4. **原文跳转**：底部固定栏，按钮样式"查看原文 ↗"

## 三、互动设计（点赞/收藏/分享）

### 3.1 竞品参考

#### 微信看一看（点赞/收藏）

**功能描述：**
- 点赞：单次点击激活，心形变红 + 数字 +1，toast 提示"谢谢点赞"

```

#### 即刻（评论/收藏）

- 评论：半屏弹窗评论输入框，带键盘自适应
- 收藏：动效丰富，图标弹跳 + 数字滚动
- 分享：生成分享卡片图，带小程序码和祝福语

## 四、小红书风格分享图生成方案

### 4.1 背景与目标

小红书分享卡片的视觉特点：
- 竖版图片（3:4 或 1:1 比例）
- 标题大字居中
- 品牌色背景 + 装饰元素
- 小程序码置于角落
- 整体精致、有质感、可传播

### 4.2 技术方案对比

#### 方案一：Canvas 2D 绘制（推荐）

**技术栈：**
- 微信 Canvas 2D API（基础库 2.9.0+，同层渲染支持）
- `wx.canvasToTempFilePath` 导出图片
- `wx.showShareImageMenu` 打开分享面板

**优点：**
- 性能高，导出速度快
- 支持同层渲染，不受原生组件限制
- 兼容性好，支持全机型

**缺点：**
- 代码量较大，需要手动绘制每个元素
- 字体加载（`wx.loadFontFace`）需要额外处理

**Canvas 绘制流程：**

```
1. wxml: <canvas type="2d" id="shareCanvas" style="width:300px;height:400px;"></canvas>
2. js: 
   - 获取 Canvas 对象和 Context
   - 设置画布尺寸（推荐 600×800px，即 3:4）
   - 绘制背景（渐变色或纯色）
   - 绘制装饰元素（圆角矩形、emoji）
   - 绘制标题文字（换行处理）
   - 绘制毒舌点评
   - 绘制小程序码（二维码）
   - 绘制 logo / 品牌标识
3. wx.canvasToTempFilePath({ canvas: canvasInstance, fileType: 'png', quality: 1 })
4. wx.showShareImageMenu({ path: tempFilePath })
```

**绘制元素参考（竖版 600×800px）：**

| 元素 | 位置 | 尺寸 | 说明 |
|------|------|------|------|
| 背景渐变 | 全画布 | 600×800 | 品牌色渐变（#FF6B6B → #4ECDC4） |
| 顶部装饰 | 顶部 40px | 600×80 | 品牌名 + 日期 |
| 主标题 | 居中偏上 | 520×200 | 最多 4 行，24pt 白色粗体 |
| 毒舌点评 | 标题下方 | 520×120 | 斜体，引号包裹，18pt |
| 分割线 | 中心 | 200×2 | 白色半透明 |
| 小程序码 | 右下角 | 120×120 | 小程序码图片 |
| 品牌 logo | 左下角 | 60×60 | 品牌标识 |
| 二维码提示文字 | 小程序码上方 | 120×30 | "今天毒什么" |

#### 方案二：北兔海报（第三方库封装）

**技术栈：**
- 社区开源方案（如Painter、mp-canvas 等）
- JSON 配置绘制模板

**优点：**
- 上手快，配置化

**缺点：**
- 维护不稳定
- 性能差于原生 Canvas 2D
- 复杂模板兼容性差

**推荐：不采用，建议自研 Canvas 2D**

#### 方案三：wxml-to-canvas（截图方案）

**技术栈：**
- 将 WXML 结构渲染到 Canvas
- 适合简单静态模板

**缺点：**
- 不支持动态数据实时渲染
- 性能一般

**推荐：仅用于静态模板海报（如品牌介绍页）**

## 五、数据字段规范建议

### 5.1 资讯内容数据模型（News Item）

```json
{
  "id": "string (UUID)",
  "title": "string (1-100 chars)",
  "summary": "string (1-300 chars)",
  "toxicComment": "string (毒舌点评，1-150 chars)",
  "content": "string (可选，详情页正文)",
  "coverImage": "string (URL，头图)",
  "source": "string (来源，如'机器之心')",
  "sourceUrl": "string (原文链接)",
  "category": "string (分类：tech/ai/startup/design)",
  "tags": ["array of strings"],
  "publishedAt": "ISO8601 timestamp",
  "readTime": "number (分钟)",
  "hotness": "number (0-100，热度值)",
  "stats": {
    "views": "number",
    "likes": "number",
    "bookmarks": "number",
    "shares": "number"
  },
  "isPinned": "boolean (是否置顶)",
  "createdAt": "ISO8601 timestamp",
  "updatedAt": "ISO8601 timestamp"
}
```

### 5.2 分类元数据（Category）

```json
{
  "id": "string",
  "name": "string (如'科技日报')",
  "slug": "string (如'tech')",
  "icon": "string (emoji 或 icon URL)",
  "color": "string (HEX 色值)",
  "description": "string (可选)",
  "order": "number (排序)"
}
```

### 5.3 用户行为数据（User Action）

```json
{
  "openid": "string",
  "likedItems": ["array of item IDs"],
  "bookmarkedItems": ["array of item IDs"],
  "readHistory": [
    {
      "itemId": "string",
      "readAt": "timestamp"
    }
  ],
  "preferences": {
    "favoriteCategories": ["array of category slugs"],
    "notificationsEnabled": "boolean"
  }
}
```

### 5.4 字段规范说明

| 字段 | 必填 | 格式规范 | 备注 |
|------|------|---------|------|
| id | ✅ | UUID / Snowflake | 全局唯一标识 |
| title | ✅ | 1-100 字符 | 建议 20-40 字 |
| summary | ✅ | 1-300 字符 | 建议 50-100 字 |
| toxicComment | ✅ | 1-150 字符 | 品牌核心亮点 |
| coverImage | 推荐 | URL，比例 16:9 或 3:2 | 建议 750×500px |
| source | ✅ | 来源名称 | 如"机器之心" |
| sourceUrl | ✅ | URL | 原文链接 |
| category | ✅ | 分类 slug | tech/ai/startup/design |
| tags | 可选 | 字符串数组 | 最多 5 个 |
| publishedAt | ✅ | ISO8601 | 发布时间 |
| readTime | 可选 | 数字（分钟） | 自动计算或手动填 |
| hotness | 可选 | 0-100 | 用于排序 |

## 六、新手引导 / 空状态设计

### 6.1 竞品参考

#### 微信读书新手引导

**首次打开：**
1. 闪屏页 → 品牌 Logo 渐显（1.5s）
2. 3 页引导轮播（左右滑动）
   - 第 1 页：产品介绍 + 毒舌风格的欢迎语
   - 第 2 页：功能介绍（如何阅读、如何毒舌点评）

3. 跳过按钮（右上角）

**特点：**
- 轮播页使用真实内容截图
- 最后一页"开始使用"按钮颜色突出
- 引导完成后存储状态，不再显示

#### 每日环球时报（新闻类）

**空状态设计：**
- 网络错误：灰色插画 + "网络开小差了" 文案 + 重试按钮
- 无内容：插画 + "今天的毒已经放完，明天再来" 文案
- Loading 状态：骨架屏（Skeleton Screen）

### 6.3 每日更新提示

**触发时机：** 每日首次打开小程序时

**提示形式：** 顶部横幅（Slide Down Banner）

```
┌─────────────────────────────────────────────┐
│  🌟 今日更新：5 条新毒闻已到                  │
│                         [立即查看] [×]      │
└─────────────────────────────────────────────┘
```

**横幅规格：**
- 高度：80rpx
- 背景：渐变 #FF6B6B → #FF8E53
- 文字：白色 14pt
- 自动消失：3 秒后自动收起（带 Slide Up 动画）
- 可手动关闭：点击 × 收起

**订阅消息推送（可选）：**

- 使用 `wx.requestSubscribeMessage` 申请订阅
- 推送模板：每日日报更新提醒
- 文案示例："🔥 今日毒闻已更新，快来领教今天的毒！"
- 限制：用户主动订阅才可推送

## 七、总结：升级建议优先级

### P0（核心体验，必做）

1. **卡片 UI 改版**：统一信息层级，毒舌点评突出显示
2. **详情页重构**：顶部大图 + 内容排版 + 毒舌高亮块
3. **分享海报功能**：Canvas 2D 绘制 + `wx.showShareImageMenu`
4. **空状态骨架屏**：替换当前 loading 效果

### P1（提升体验，建议做）

1. **新手引导轮播**：3 页介绍品牌 + 功能
2. **每日更新横幅**：首访更新提醒
3. **收藏功能优化**：动效 + Toast 反馈
4. **数据字段补充**：hotness、readTime 等字段

### P2（增值功能，择机做）

1. **订阅消息推送**：每日日报提醒
2. **分享朋友圈海报**：小红书风格竖版
3. **用户阅读历史**：本地存储 + 同步
