## 五、流程改进（固化）

### 新规则 1：发版前必走校验官
```
Stephen 改代码 → 校验官审查 → 通过 → Lucia 发版
                                    → 打回 → Stephen 修 → 重新送审
```

### 新规则 2：产品方案先于代码
```
Boss 提需求 → Thomas 出产品方案（15min 轻量级）→ 确认 → Stephen 写代码
```
禁止：需求直接跳代码。

### 新规则 3：CSS 精准修改原则
- 不写全量选择器（`view { ... }`）
- 修改最小范围，用 class 选择器
- 改完自己测一遍 iPhone 模拟器

### 新规则 4：Cron 错峰
```
3:10 industry-daily
3:15 dev-daily
3:20 startup-daily
3:25 ai-daily
3:30 design-daily
3:35 signals (hn, github, arxiv)
```
间隔 5 分钟，避免并发 token 竞争。
