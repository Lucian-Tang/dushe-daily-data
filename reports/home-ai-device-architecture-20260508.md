#### 麦克风阵列 — ReSpeaker 4-Mic USB 版
```
ReSpeaker 4-Mic          树莓派 5
┌────────────────┐      ┌──────────┐
│  4-Mic Array   │──USB──→│ USB 3.0  │
│  (UAC2.0)      │       │  (声卡)  │
│                │ I2S   │          │
│  3.5mm Audio   │───→───│ GPIO 18  │
│     Out        │  PWM  │ (PWM音频) │
└────────────────┘      └──────────┘

GPIO 分配 (I2S 音频):
- GPIO 18 → I2S Data Out (播放)
- GPIO 19 → I2S Data In  (录音)
- GPIO 20 → I2S Word Select (声道切换)
- GPIO 21 → I2S Bit Clock  (时钟)
- GND    → 共地

注意：ReSpeaker USB 版本无需 GPIO接线，即插即用，
      内部已通过 USB 虚拟为 UAC2.0 声卡。
```

### 4.1 语音对话完整链路

```
用户: "Hey Lucia，今天天气怎么样？"
│
│ [阶段 1: 唤醒词检测]
│ 耗时: ~50ms
│
│  ┌─────────┐    麦克风持续收音     ┌──────────────────┐
│  │ReSpeaker│ ←──────────────────── │  Picovoice       │
│  │ 4-Mic   │    (VAD 流)           │  Porcupine       │
│  └────┬────┘                       │  (本地唤醒引擎)   │
│       │  音频流 (16kHz PCM)        └────────┬─────────┘
│       │                                 唤醒词命中
│       ▼                                    ▼
```

### 4.2 延迟预算详解

```
┌──────────────────┬────────────┬─────────────────────────────────────────────┐
│      阶段        │  典型延迟   │              说明                            │
├──────────────────┼────────────┼─────────────────────────────────────────────┤
│ 唤醒词检测        │   30-80ms  │  本地，实时流处理                           │
│ 语音结束检测(VAD) │  200-500ms │  等待用户说完再发送                         │
│ 音频传输          │   20-50ms  │  局域网 WiFi 5GHz / 以太网                 │
│ ASR (Whisper)    │  300-600ms │  Mac Mini M4 GPU 加速，中文稍慢              │
│ LLM 首 token     │ 2000-4000ms│  Ollama + Llama 3.3 70B，Mac Mini M4 24GB   │
│ LLM 生成长度     │ ~30ms/token │  70B Q4 量化约 25-35 tokens/s              │
│ TTS 预热         │  300-500ms │  XTTS 首次生成需要预热                      │
│ TTS 流式生成     │  实时流     │  边生成边传输，约 0.5x 实时                 │
│ 音频播放缓冲     │   50-100ms │  最小缓冲避免卡顿                           │
│ 屏幕更新         │   50-100ms │  UI 线程异步更新                            │
├──────────────────┼────────────┼─────────────────────────────────────────────┤
│ 端到端总计       │ 4000-8000ms│  首次响应（含完整 LLM 生成）                 │
│ (不含用户语音时长)│            │                                             │
└──────────────────┴────────────┴─────────────────────────────────────────────┘

优化空间:
  • LLM 首 token 延迟：用更小的模型（Llama 3.1 8B）可降至 ~500ms
  • TTS 预热：用 Keep-alive ping 每 5s 保持模型热状态
  • VAD 延迟：调整静默检测阈值为 500ms 而非 700ms
```
