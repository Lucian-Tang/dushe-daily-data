# Luxmind 技能价值评估报告

**评估人：** Thomas（产品经理）
**日期：** 2026-05-09
**产品线：** 今天毒什么（毒舌日报）、洞见（品味挑战）、信号采集系统

---

## 技能评估

### 1. gzh-prohibited-word — 公众号违禁词检测

| 维度 | 评估 |
|------|------|
| **产品价值** | 信号采集系统：采集公众号内容时自动过滤违规风险，提升内容安全性 |
| **优先级** | **P1** |
| **理由** | 内容合规是公众号生态红线，提前拦截可避免毒舌日报踩雷 |

---

### 2. xhs-prohibited-word — 小红书违禁词检测

| 维度 | 评估 |
|------|------|
| **产品价值** | 信号采集系统：小红书是重要信号源，违禁词检测保障内容可用性 |
| **优先级** | **P1** |
| **理由** | 小红书是毒舌日报的重要信息来源，违禁词过滤是内容质量保障链的一环 |

---

### 3. baidu-doc-pipeline-parser — 百度文档解析

| 维度 | 评估 |
|------|------|
| **产品价值** | 信号采集系统：扩展采集能力至百度文库等文档源，丰富信号库 |
| **优先级** | **P2** |
| **理由** | 文档类信号源扩展，优先级低于结构化媒体采集，但能提升内容覆盖广度 |

---

### 4. baidu-doc-vlm-parser — 百度PaddleOCR-VL大模型文档解析

| 维度 | 评估 |
|------|------|
| **产品价值** | 信号采集系统：提升图片/扫描件等非结构化文档的解析精度 |
| **优先级** | **P2** |
| **理由** | 图像类文档解析能力，是 baidu-doc-pipeline 的高精度替代方案，ROI 待验证 |

---

### 5. gangtise-file — 更知文件中心检索

| 维度 | 评估 |
|------|------|
| **产品价值** | 信号采集系统：获取研报/公告/纪要等结构化金融文档，补充高质量信号源 |
| **优先级** | **P1** |
| **理由** | 研报和公告是高质量信号源，与毒舌日报/洞见的深度内容需求高度匹配 |

---

### 6. bot-file-processor — 通用文件处理

| 维度 | 评估 |
|------|------|
| **产品价值** | 支撑信号采集系统：批量格式转换/重命名，让采集的原始文件可直接消费 |
| **优先级** | **P1** |
| **理由** | 解决采集后的文件规范化问题，属于基础设施层，值得优先建设 |

---

### 7. kg-note-method-obsidian — Obsidian KG笔记法整理

| 维度 | 评估 |
|------|------|
| **产品价值** | 内部知识管理：辅助产品团队整理信号笔记，与 Luxmind 知识管理契合 |
| **优先级** | **P2** |
| **理由** | 对外的用户价值有限，主要是内部效率工具，不直接服务于产品体验 |

---

### 8. 96push — 96Push 桌面端发布

| 维度 | 评估 |
|------|------|
| **产品价值** | 今天毒什么（毒舌日报）：多平台内容分发，扩大毒舌日报传播渠道 |
| **优先级** | **P0** |
| **理由** | 直接赋能内容分发，是毒舌日报扩大影响力的核心路径，P0 优先级最高 |

---

### 9. qwen-web-agent — 通义千问 Web 浏览器自动化

| 维度 | 评估 |
|------|------|
| **产品价值** | 信号采集系统：自动化采集动态/登录类页面，大幅扩展信号源覆盖 |
| **优先级** | **P0** |
| **理由** | 解决复杂页面的自动化采集问题，是信号采集系统的能力倍增器 |

---

## 优先级汇总

### 🔥 P0（必须投入）

| 技能 | 理由 |
|------|------|
| **96push** | 内容分发是毒舌日报的核心出口，决定产品影响力 |
| **qwen-web-agent** | 解决动态页面采集瓶颈，是信号采集系统的能力跃升关键 |

### ⚡ P1（建议投入）

| 技能 | 理由 |
|------|------|
| **gzh-prohibited-word** | 公众号内容合规过滤，日报安全底线 |
| **xhs-prohibited-word** | 小红书信号源保障，日报内容安全 |
| **gangtise-file** | 研报/公告高质量金融信号，匹配毒舌日报深度需求 |
| **bot-file-processor** | 文件规范化基础设施，提升采集效率 |

### 📌 P2（可延后）

| 技能 | 理由 |
|------|------|
| **baidu-doc-pipeline-parser** | 文档源扩展，优先级低于媒体采集 |
| **baidu-doc-vlm-parser** | 高精度图像解析，ROI 待验证 |
| **kg-note-method-obsidian** | 内部知识管理，外部用户价值有限 |

---

## 建议

**短期（1-2个月）：** 先落地 **96push** + **qwen-web-agent**，这两项直接扩大内容分发和采集覆盖面，是最直接的增长杠杆。

**中期：** 接入 **gangtise-file** 补充高质量研报信号，同步建设 **bot-file-processor** 规范化基础设施，辅以 **gzh/xhs-prohibited-word** 合规防线。

**长期：** 文档解析类技能可根据信号采集的实际缺口再评估引入必要性。
