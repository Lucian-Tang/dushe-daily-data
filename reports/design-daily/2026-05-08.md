## 🛠 设计工具 / 新产品

**1. Sendly**
来源：Product Hunt
点评：给AI Agent和开发者用的短信服务——SMS for AI Agents。说白了就是让大模型能打电话/发短信，听起来很性感但总觉得哪里不对，AI已经有自己的通讯方式了，还要SMS干嘛？🤔

**2. Kuku: open source**
来源：Product Hunt
点评：本地化的"第二大脑"，为所有AI服务的开源知识管理工具。说人话就是：你的私人AI需要一个好用的笔记本，而这个是完全离线的，数据全在自己手里。隐私党狂喜。

**3. APIEval-20**
来源：Product Hunt
点评：测试AI Agent调用API能力的开源基准测试。说实话有点卷，但意义重大——以后吹牛说自己Agent多强，得先在这套基准上跑个分。

## 🎨 设计作品趋势

**4. Smashing Magazine: The Architecture Of Local-First Web Development**
来源：Smashing Magazine
点评：本地优先架构在2026年终于从学术圈走向工程实践了。作者用自己的血泪史（里斯本酒店demo翻车）讲清楚了local-first不是离线优先，是数据主权归用户。这个认知差价值百万。

**5. Figma + Claude Code 设计系统工作流指南**
来源：UX Collective
点评：AI写Figma代码看起来很美，但一进图层就原形毕露——颜色不用design token，组件freestyle。这篇文章给出了实战解法：Preflight检查 + Reference Interpreter + Component Rules + Style Binding，四步让AI乖乖用你的设计系统。

## 📐 设计方法论 / 趋势

**6. Design System Governance**
来源：UX Collective
点评：设计系统最难的从来不是建，而是"管"。变量、Token、命名规范——这些 governance 层面东西比组件库本身重要十倍。你的设计系统缺的不是按钮，是规则。

**7. AI Agent UI 的设计挑战**
来源：综合趋势
点评：大模型对话界面是新的设计荒漠。传统UI范式（按钮、表单、菜单）正在被"对话+Agent执行"取代，但没人知道Agent完成一个任务需要多少步、失败时如何干预、用户何时该介入。这块设计研究严重滞后。

## 🌟 值得关注的设计案例

**8. Fabraix**
来源：Product Hunt
点评：在AI Agent发布前帮你在真实场景里找到gap——说人话就是给Agent做"压力测试"。思路很对，大模型缺的往往不是智商，是边界意识。

**9. GitHired 2**
来源：Product Hunt
点评：用作品证明能力而非简历关键词找工程师。设计师同理：你的Dribbble Behance比LinkedIn有说服力多了。这类产品在挑战传统的"资历歧视"。
