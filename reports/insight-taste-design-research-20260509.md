# 琢光 · 品味感设计研究报告

> 作者：Lucia
> 日期：2026-05-09
> 状态：设计研究 v1.0

## 一、竞品设计语言分析

### 1. Linear — 极简的终极形态

Linear 是软件界的 Apple——每个像素都有理由存在。

色彩：深黑底色（#0D0D0D）+ 单一强调色（紫色系 #5E6AD2）+ 极少第三色。无渐变，无阴影泛滥。
字体：Inter，仅用 Regular/Medium/Semibold 三个字重。大小层级严格：标题/正文/辅助文字比例 1.5:1:0.75。
动效：所有转场都是 spring 曲线，有惯性但不过度。列表展开是真正的"生长"感而非闪出。
空间：间距体系基于 4px 网格，最小单位 4px。
独特性：**克制到极致。** 不做任何"华丽"的事情，因为觉得不需要。

借鉴点：琢光的背景色保持极暗，强调色只用一种黄金色，不做多余装饰。

### 2. Duolingo — 游戏化不油腻

Duolingo 证明了"有趣"和"廉价"之间的那条线。

色彩：鲜艳的绿色主调（#58CC02）+ 高饱和辅色。但关键在暗黑模式下会降 20% 饱和度。
字体：Feather Bold（品牌字体）+ DIN Next Rounded。圆角字型 + 粗体让"学习"没有压迫感。
动效：每个正确的动画都是一次小庆祝——但庆祝完立刻进入下一题，不停留。失败动画是抖动而非弹窗。
空间：卡片圆角 16px，元素间大间距，绝不让信息挤在一起。
独特性：**用动画制造多巴胺，但从不打断心流。**

借鉴点：答题正确/错误的反馈动画要有"奖励感"但不打断节奏。

### 3. Headspace — 宁静的力量

Headspace 的所有设计决策都在说一个词："不着急"。

色彩：暖橘主调（#F68B33）+ 大量暖白/米白色空间。没有纯黑，最深的是深蓝灰。
字体：圆形字型，标题是手写体感的粗体。中文用圆体系列。
动效：所有动画都是缓入缓出，时长 400-600ms。圆形"呼吸"动画贯穿全局。
空间：大量留白。一张屏只放一个核心信息。辅助信息收缩到边缘。
独特性：**空间就是内容。** 留白不是在浪费屏幕，是在建立信任。

借鉴点：鉴赏页面的艺术品展示——一张屏一幅作品，让用户沉浸，不被 UI 打扰。

### 4. Endel — 声音可视化

Endel 把"听不清的东西"做得极致可见。

色彩：渐变是核心——从一种氛围色过渡到另一种，没有生硬色块边界。
字体：极轻的 sans-serif，字号很小，让位给视觉。
动效：持续的低频呼吸动画，像心跳一样，绝不停歇。
空间：全屏沉浸式，UI 几乎不动地方，所有交互靠长按和滑动。
独特性：**UI 是为了消失而存在的。**

借鉴点：语音交互界面的设计——不要显眼的录音按钮，用呼吸动画代替。

### 5. Apple Music / Spotify — 内容为王

两个音乐 App 的共同哲学：内容是主角。

色彩：Apple Music 用纯白/纯黑+专辑色提取。Spotify 用深黑+品牌绿。共同点是——专辑封面在哪，色彩就跟到哪。
字体：Apple 用 SF Pro Display 大字重标题。Spotify 用 Circular 品牌字。
动效：专辑封面放大转场是核心动画。列表滚动跟手性极好。
空间：封面图占 60%+ 视觉面积，文字信息不超过 3 行。
独特性：**让内容发光，不抢内容的风头。**

借鉴点：鉴赏内容展示——图片占主导，文字做陪衬，最大化沉浸感。

### 6. Artsy / DailyArt — 艺术品展示的范本

Artsy 是世界上最懂"怎么展示一幅画"的产品之一。

色彩：始终白色/深灰背景，因为"背景不该和画抢注意力"。
字体：Serif 标题（Garamond/Caslon）+ Sans-serif 辅助。艺术品用衬线字体天然显得有学识。
动效：几乎没有动效。图片加载用渐显，不做飞入飞出。
空间：画作占屏幕 80% 宽度，上下各留 40-60px 白边。画作信息不叠在画上。
独特性：**"不打扰"是最高级的尊重。**

借鉴点：琢光鉴赏页 = 画作居中 + 极简描述 + 留白呼吸。

### 7. Notion — 信息架构的优雅

Notion 证明了"功能多"和"看着乱"是两回事。

色彩：浅灰背景+极淡卡片。强调色几乎不可见，因为"工具不该有性格"。
字体：Inter。所有文本层级只用字号和灰度区分，不做加粗滥用。
动效：拖拽排序的物理感。块级菜单 hover 才出现——需要的时候才在，不需要就消失。
空间：左侧栏固定宽度，内容区自适应。信息密度可控。
独特性：**UI 只在需要的时候出现。**

借鉴点：功能多的页面（如训练分类）——信息架构要呼吸，不是堆砌。

### 8. Carrot Weather — 有性格的幽默设计

Carrot 证明了"有趣"和"品味"可以共存——只要幽默本身有品味。

色彩：天气驱动的动态渐变（晴=暖色，雨=冷色）。
字体：粗体无衬线，配合"AI 吐槽"的语气。
动效：天气动画是重点。UI 本身的动效克制。
空间：天气动画占上半屏，数据在下半屏。
独特性：**AI 人格通过 UI 表达出来。** 不只是"好看"，是"有个性"。

借鉴点：琢光的毒舌人格可以通过微文案和过渡动画表达，不用改设计语言。

## 二、琢光专属设计原则

### 2.1 色彩体系

**主色调：暗黑基底 + 黄金强调**

深黑基底：`#0A0A12`（比纯黑多一丝蓝，更像夜空而非黑屏）
卡片层：`#111122`（比基底高 5% 亮度，有层次但不跳）
强调色：`#FFD700`（黄金色，克制使用，只用在一屏一个核心元素）
辅助强调：`#C9A84C`（暗金，用于次要强调）
成功色：`#4ADE80`（翠绿，非荧光绿）
错误色：`#E85D75`（暖红，不刺眼）
文字色：`#E8E8E8`（主要）/ `#888888`（次要）/ `#555555`（三级）

**阶梯度使用规则：**

强调色占比不超过页面面积 5%
一个页面最多 2 处使用强调色
大面积颜色只来自图片内容本身

### 2.2 字体层级

**中文主字体：PingFang SC / 苹方**

大标题：36-48rpx，Semibold（W6），用于页面主标题，极少使用
中标题：28-32rpx，Medium（W5），用于模块标题
正文：26-28rpx，Regular（W4），用于内容描述
辅助文：22-24rpx，Regular（W4），用于标签、提示
微文：20rpx，Regular（W4），用于时间/数据

**字体规则：**

一屏最多 3 种字号
不在同一屏混用加粗和正常体——要么都粗要么都细
数字用 Tabular Figures（等宽数字），让数据对比整齐

### 2.3 动效哲学

**什么该动：**

值得庆祝的时刻（答对一题 / 段位升级）→ 200-300ms spring 动画
内容转场（卡片切换到下一张）→ 300ms ease-out 滑动
数据变化（分数上涨 / 进度条推进）→ 400-600ms ease 计数动画
加载状态（内容未就绪）→ 呼吸动画或骨架屏

**什么不该动：**

页面切换：禁用默认的从右滑入，用 150ms 渐显
列表加载：不做交错入场动画
按钮点击：不做缩放回弹（除非是"提交答案"这种确认操作）
背景：不做自动轮播，不做视差

**核心原则：** 动画是标点符号，不是正文。少即是多。

### 2.4 空间与留白

**基于 8rpx 网格体系：**

最小间距：8rpx（元素内部）
标准间距：16rpx（相关元素间）
模块间距：24-32rpx（模块间）
页面边距：32rpx（左右安全区）

**留白规则：**

鉴赏内容页：图片占 60-70% 屏高，上方不留 UI，下方文字区距图片至少 40rpx
名片页：徽章区域占 35% 屏高，周围留白至少 60rpx
答题页：题目区占 40% 屏高，选项间距 16rpx，上距至少 40rpx
列表页：卡片高度 ≥ 120rpx，卡片间距 12rpx

### 2.5 微交互规范

**滑动：** 鉴赏内容上下滑动，跟手系数 0.95（略微阻尼，不飘）
**点击：** 卡片点击缩小到 97%，100ms 回弹到 100%
**长按：** 语音录制触发，配合触觉反馈（wx.vibrateShort）
**转场：** 页面间用 150ms crossfade，不用推入推出

## 三、关键页面品味 Checklist

### 今日首页

状态：最常被看到的页面，第一印象
品味要求：不能堆砌信息，不能像"功能大厅"

Checklist：
- 顶部只说一件事：今天的鉴赏内容，不言自明
- 鉴赏图片占 60% 屏高，文字渐变叠加在图片底部
- 三个反馈按钮在图片下方，间距宽松，不做任何视觉抢夺
- 今日挑战入口用一张安静卡片，不放🔥这种催促性 emoji
- 段位卡片精简到一行：emoji + 名称 + 进度条
- 底部品牌区极简：一条金线 + "Luxmind" 一行字

### 鉴赏内容展示页

状态：品味的核心战场，怎么做直接定义品牌调性
品味要求：每一条内容都要有"值得看"的感觉，不能像信息流刷屏

Checklist：
- 一张屏只展示一件作品
- 图片占 65-70% 高度，底部留出文字区的自然呼吸
- 文字层级：来源（最小）→ 标题（最大）→ 描述（正文）→ 琢光栏（有颜色区分）
- 琢光栏用左边框强调色 + 淡色背景
- 语音评论入口不做显眼的"录音按钮"，用底部呼吸动画提示
- 上下滑动切换内容，像翻画册而非刷信息流

### 答题页

状态：判断训练的核心交互
品味要求：不能像考试，不能像问卷，要有"智力快感"

Checklist：
- 题目文字大而清晰，背景极暗，文字浮在"虚空"中
- 选项卡片间距充足，未选中时低调，选中后金色描边
- 正确反馈：绿色微光 + 短暂停留（400ms）后自动下一题
- 错误反馈：红色微光 + 正确答案高亮 + 解析文字渐进显示
- 进度条在顶部，极细，不抢注意力
- 不做"倒计时"——品味不是速度比赛

### 名片页

状态：社交身份的表达，用户会截图分享
品味要求：高级、克制、不像游戏战绩

Checklist：
- 段位徽章用圆环 + emoji，不做复杂的 3D 或投影
- 能力维度用细进度条，颜色低调，数值比进度条更显眼
- Vibe 标签用圆角小标签，2-3 个最佳，不做标签墙
- 战绩数字用大号等宽字体，不加单位（"7次"不如"7"干净）
- 分享按钮用实心金按钮，不需要"优惠"或"限时"话术
- 底部进化史预告用模糊卡片，暗示"还有更多"

### 语音交互界面

状态：未来核心交互，但现在只是设计占位
品味要求：不显眼，不打断，自然融入

Checklist：
- 录音入口不用红色大圆点（那是"正在录制"的标志，不是"请录音"的标志）
- 用底部浮层 + 呼吸动画 + 提示文字 "按住说话"
- 录音中：一个圆环呼吸动画，不显示时长数字
- 录音完成：轻微震动 + 快速显示文字转写
- 不做语音波形可视化——那是功能展示，不是品味

## 四、反模式：什么不能做

### 🔴 绝对禁止

亮色/白色背景：琢光不是笔记工具，暗色是品牌识别的根基
荧光色：用 #00FF00 这种高饱和霓虹色直接掉价到山寨App水平
大面积渐变按钮：渐变做强调色可以，但整屏渐变 = 杀马特
弹窗广告：任何形式的弹窗推广都会毁掉品味感
红点/未读标记：品味不需要 FOMO
"限时优惠""最后3天"：品味产品不制造焦虑
进度条玩数字游戏："你还差1分升级"——太低级的留存手段
堆砌 emoji：每屏最多 2 个 emoji，不做 emoji 墙
第三方廉价插图：不用矢量插画，要么真实作品图，要么纯文字
无意义的动效：飞入、弹跳、旋转——这些是 PPT 不是品味产品

### 🟡 谨慎使用

成就弹窗：有庆祝但不是打断（答对后轻触反馈比弹窗好）
排行榜：可以存在但不要放在显眼位置——品味不是排名
用户生成内容：先做专业策展再开 UGC，品质优先
社交功能：先做好个人体验，好友功能后置

## 五、设计落地优先级

**Phase 1（本周）：**
- 暗黑基底色统一到 #0A0A12
- 强调色统一到 #FFD700，去除页面中的杂色
- 字体层级对齐（PingFang，三种字号以内）

**Phase 2（Phase 1 完成后）：**
- 鉴赏内容页排版重构（画作居中+琢光栏设计）
- 答题页正确/错误动画
- 名片页视觉升级

**Phase 3（语音上线时）：**
- 语音交互界面设计
- 微交互动效系统
