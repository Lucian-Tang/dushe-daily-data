## 2. iOS App 架构

### 2.1 技术选型：SwiftUI vs UIKit

```
┌──────────────────────────────────────────────────────────────────────────────┐
│                          iOS 架构技术选型对比                                   │
├────────────────────┬─────────────────────┬──────────────────────────────────┤
│        维度         │      SwiftUI        │            UIKit                │
├────────────────────┼─────────────────────┼──────────────────────────────────┤
│ 声明式 vs 命令式    │ 声明式（推荐）        │ 命令式                            │
│ 开发效率            │ 高（预览即时生效）     │ 中（编译+运行周期长）             │
│ 状态管理            │ @State / @Observed   │ Delegate / NotificationCenter   │
│ 跨设备适配          │ 声明式布局，天然适配   │ Auto Layout，约束复杂             │
│ 动画系统            │ withAnimation 简洁   │ UIView.animate                   │
│ 系统 API 访问        │ 需通过 UIViewRepresentable│ 直接访问摄像头/麦克风 API     │
│ 语音/视频捕获       │ AVCaptureSession 封装│ 直接使用，较成熟                  │
│ 第三方库生态        │ 相对较少              │ 丰富（大量 CocoaPods/SPM）        │
│ 维护成本            │ 低（Apple 主推）      │ 中（历史包袱）                    │
└────────────────────┴─────────────────────┴──────────────────────────────────┘
```

**推荐理由：**
- **SwiftUI** 用于所有 UI 层面，享受声明式开发效率与实时预览
- **UIKit** 仅用于感知采集层（AudioCaptureManager / VideoCaptureManager），直接封装 AVCaptureSession 和 Vision 框架
- 两者通过 `UIViewRepresentable` / `UIViewControllerRepresentable` 桥接

```
┌──────────────────────────────────────────────────────────────────────────────┐

├──────────────────┬───────────────────────────────────────────────────────────┤
│      方案         │                     说明                                 │
├──────────────────┼───────────────────────────────────────────────────────────┤
│                  │  Apple 官方语音识别框架                                    │
│  Speech          │  • SFSpeechRecognizer: 持续监听，本地运行                   │
│  Framework       │  • 可定义自定义唤醒词短语                                   │

│                  │  • 延迟: ~50-100ms                                        │
├──────────────────┼───────────────────────────────────────────────────────────┤
│                  │  Apple Siri 唤醒                                          │
│  SiriKit /       │  • "Hey Siri" 硬件级支持，iPad 原生唤醒                    │
│  Siri            │  • 优点: 零功耗待机监听，Siri 级别唤醒                       │
│                  │  • 缺点: 必须走 Siri 生态，无法自定义唤醒词                  │
│                  │          对话内容受 Apple 审核限制                          │
│                  │  • 适用场景: 简单指令（"Hey Siri 开灯"）                   │
├──────────────────┼───────────────────────────────────────────────────────────┤
│                  │  Picovoice Porcupine (跨平台)                              │
│  Porcupine       │  • 唤醒词完全本地运行，iPad 可用                           │
│  (第三方)        │  • 支持中文唤醒词自定义                                     │
│                  │  • 缺点: 需要集成第三方库，商业使用付费                     │
└──────────────────┴───────────────────────────────────────────────────────────┘

推荐方案：Speech Framework（本地）+ Porcupine（备选中文唤醒词）

混合策略：
  1. App 进入后台 → 使用 Siri 被动唤醒（节省本地资源）
  2. App 在前台 → 使用 Speech Framework 持续监听（更灵活）
  3. 自定义唤醒词（如"小 Lucia"）→ Porcupine
```
