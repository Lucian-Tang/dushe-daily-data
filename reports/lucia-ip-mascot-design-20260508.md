# Lucia IP 形象设计方案

> 设计者：Thomas

> 目标：小程序顶部导航区域 IP 化

---

## 1. 动物选型分析

### 候选 A：狐狸 🦊

**优势：**
- 狐狸本身有「机灵、狡黠」的气质，与 Lucia 毒舌人设天然契合
- 颜色鲜明（橙色系），在界面上容易出挑
- 狐狸在中国文化里有一定认知度，但不是太常见的 IP 动物，记忆点强
- 「尾巴」可以作为情绪变化的天然载体（翘起/垂下/炸毛）

**劣势：**
- 狐狸有时被与「狡猾、奸诈」绑定，需要在设计上调教成「毒舌萌」而非「腹黑」
- 狐狸形象在中文互联网有「狐狸精」等偏负面语境，需要稍微正面引导

**匹配度：⭐⭐⭐⭐**

---

### 候选 B：猫 🐱

**优势：**
- 猫的「慵懒、傲娇」性格与 Lucia 的「嘴硬心软」高度一致
- 表情变化直观（眯眼、竖耳、甩尾），用户感知门槛低
- 中国用户对猫的接受度极高，宠物化情感强
- 设计素材多，社区熟悉度高

**劣势：**
- 猫的 IP 形象实在太多了（HelloKitty、吾皇、阿秃……），差异化难度大
- 「傲娇」和「毒舌」有一定重叠，但如果无差异化处理，容易流于平庸
- 猫的「慵懒」感如果太强，可能让产品显得缺乏能量

**匹配度：⭐⭐⭐**

---

### 候选 C：猫头鹰 🦉

**优势：**
- 「睿智、洞察」—— Lucia 懂科技、毒舌有深度，猫头鹰能传达「我知道你在想什么」
- 眼睛大而亮，适合做「发光」等状态变化
- 在中国 IP 化程度低，差异化潜力大
- 夜晚活动与「新闻聚合」场景有天然契合（夜间浏览新闻）

**劣势：**
- 猫头鹰在中国文化里情感联结较弱，不像猫/狗有宠物情感基础
- 「呆萌」方向如果拿捏不好，容易显得「冷」
- 设计成本高，需要重新建立用户认知

**匹配度：⭐⭐⭐**

---

## 🏆 最终推荐：**狐狸**

**理由：**
1. **人设契合度最高** —— 狐狸的「机灵狡黠」完美承载毒舌人格，比猫更有个性，比猫头鹰更有温度
2. **视觉差异化** —— 狐狸在新闻/内容类产品中较少出现，比猫更有辨识度
3. **情绪载体丰富** —— 尾巴+耳朵+眼神三元素组合，能撑起 5+ 种状态的视觉表达
4. **中国用户接受度** —— 狐妖、狐狸在网络文化中是正面的萌系形象，大众接受无门槛

## 2. 形象描述

### 外观特征

| 元素 | 设计细节 |
|------|----------|
| **主色** | 橙红色渐变 (#FF6B35 → #F7931E)，暖调有活力但不俗气 |
| **体态** | 侧身坐姿，露出上半身，类似日系 Q 版 mascot 风格 |
| **耳朵** | 尖耳，内侧浅粉色，左耳略歪（俏皮感） |
| **眼睛** | 琥珀色大眼，瞳仁有高光，眼尾微微上挑 = 毒舌感 |
| **表情** | 平时微微眯眼，嘴角上扬 —— 「我什么都知道但我不想说」 |
| **尾巴** | 大而蓬松的橙色尾巴，盘在身侧，情绪指示器 |
| **服饰** | 基础款卫衣（深灰/黑），领口露出脖子，简笔但有质感 |
| **标志性元素** | 头顶一缕翘毛 + 左手腕一个小型智能手环（暗示科技感） |

### 性格标签

> **毒舌 / 机灵 / 傲娇 / 暖心 / 科技感**

### 一句话 Slogan

> **「我知道今天的热点在哪，但我不随便告诉你。」**

## 3. 状态变化系统

### 5种状态 + 对应姿态

| 状态 | 时间区间 | 姿态描述 | 视觉关键词 |
|------|----------|----------|------------|
| ☀️ **晨间模式** | 06:00–09:00 | 半睁眼，打哈欠，一撮呆毛立着，尾巴微微下垂 | 慵懒、迷糊 |
| 💻 **工作模式** | 09:00–18:00 | 眼睛放大，耳朵竖直，盯着屏幕状（视线偏右下），尾巴轻晃 | 专注、机灵 |
| 🌙 **晚间模式** | 18:00–23:00 | 眯眼，头微微歪，尾巴卷起来盘在身前 | 放松、犯困 |
| ⚡ **新闻触发模式** | 推送时 | 眼睛突然亮起（高光×2），耳朵前倾，尾巴炸开一点 | 兴奋、警觉 |
| 😴 **周末模式** | 周末全天 | 完全瘫躺，四仰八叉，呆毛塌了，嘴角流口水（！） | 摆烂、摸鱼 |

### 切换逻辑

```
if (weekend) → 周末模式
else if (6:00–9:00) → 晨间模式
else if (9:00–18:00) → 工作模式
else if (推送事件触发) → 新闻触发模式（临时，持续10s）
else → 晚间模式
```

## 4. 顶部布局方案

### 推荐方案：IP 占位 + 水平 Tab 条

```
┌─────────────────────────────────────────────┐
│  [Lucia IP 形象区]         [今天毒什么 ▼]   │  ← 同一行，IP 60px 高
│  （左侧，动态状态展示）     （下拉筛选）     │
├─────────────────────────────────────────────┤
│  [推荐] [科技] [社会] [娱乐] [汽车] [更多]  │  ← Tab 可横向滚动
└─────────────────────────────────────────────┘
```

### 视觉层级

| 层级 | 元素 | 优先级 | 说明 |
|------|------|--------|------|
| 第1层 | Lucia IP + 状态动画 | 最高 | 左上角，视觉锚点 |
| 第2层 | 「今天毒什么」标题 | 中 | IP 右侧，文字按钮呼出筛选 |
| 第3层 | 频道 Tab Bar | 中高 | 横向滚动，独立一行 |
| 第4层 | 标签流（热搜标签） | 低 | Tab 正下方，传统内容区 |

### IP 区细节

- **宽度**：80–100px，固定撑满高度约 80px
- **背景**：与顶部整体背景同色，不做割裂
- **Tab 位置**：单独一行，在 IP 区域下方，页面主体顶部
- **不要做的事**：不要把 Tab 放进 IP 区域里，不要让 IP 和 Tab 抢空间

## 5. 实现路径

### CSS 动画能做到什么

✅ **完全可行（CSS + 简单 PNG 切换）：**

- 眼睛眯/睁（大眼→眯眯眼）：用 `transform: scaleX()` 实现
- 耳朵竖/歪：CSS rotate 配合 transform-origin
- 尾巴晃动：CSS `@keyframes` rotate 往复动画（缓动）
- 背景微动（如尾巴轻微摇）：可用 `translateY` 循环
- 状态切换：JavaScript 控制 `data-state` 属性，CSS 读取切换动画

⚠️ **需要 PNG 资源的（精细表情）：**

- 嘴型变化（微笑→毒舌勾嘴→打哈欠）
- 眼睛「发光」效果（径向渐变叠加）
- 「周末瘫倒」的整体姿态（全身图差异大，CSS 无法模拟）

### 推荐技术方案

**方案 A（轻量）：表情 PNG 序列 + CSS 切换**
- 准备 5 套 PNG（图+遮罩），通过 `background-image` 切换
- 优点：视觉质量高，开发简单
- 缺点：需要维护一套图片资源（约 5 张 base state × 状态数）

**方案 B（纯 CSS）：CSS 动画 + SVG 内联**
- 所有元素用 SVG 或 CSS 绘制，通过 class 切换动画
- 优点：零图片资源，性能好，可动态着包
- 缺点：设计成本高，视觉精致度略低

### 推荐：方案 A（PNG 序列）为主 + CSS 微动为辅

> 原因：小程序包体积有限，单张 IP 图控制在 20KB 以内（WebP 格式），5 张状态 = 约 100KB，完全可接受。

### 资源清单

| 资源 | 数量 | 建议格式 | 单张建议大小 |
|------|------|----------|-------------|
| IP 状态图 | 5张（晨/工/晚/触发/周末） | WebP | ≤20KB |
| 各状态眼睛细节 PNG（叠加层） | 3张 | WebP | ≤5KB |
| 尾巴动画帧 | 2-3张 | WebP | ≤10KB |
| **合计** | **约 10-12 张** | | **≤150KB** |
