**版本：v20260508-redesign**

**面向：Lucia / Boss**

## 一、改版背景与目标

当前版本的问题核心：**用户进来后不知道在看什么，看完也不知道得到了什么价值。**

日报是效率工具，不是玄学产品。首页要给用户"今天我知道发生了什么"的确定感，而不是一张看不懂的雷达图加一堆没上下文的标题。

**重构目标：**
- 知道今天最重要的 3-5 件事是什么
- 能快速扫完，决定要不要点进去细看
- 毒舌是调味剂，不是负担

## 二、首页整体结构调整

### 2.1 移除情绪雷达图

**理由：** 5 维情绪（愤怒/娱乐/讨论/焦虑/正能量）对用户认知毫无帮助——我不知道"焦虑"值 72% 意味着什么，它不能帮我做决策。

**替换方案（三选一，推荐方案 B）：**

| 方案 | 描述 | 优点 | 缺点 |
|------|------|------|------|
| A. 关键词云 | 今日热词可视化，字号 = 热度 | 直观、有科技感 | 遮挡内容、占用空间 |
| **B. 热点话题标签流** | 横滑标签：#ChatGPT #AI搜索 #芯片封锁 | 轻量、占用少、用户秒懂 | 无个性化 |
| C. 今日概览一句话 | "今日科技圈大事：AI 模型军备竞赛持续加码" | 节省空间、有态度 | 生成成本高 |

**推荐方案 B：热点话题标签流**

实现方式：
- 固定一行横滑标签（高度 36px），每次取热度最高的 8-10 个话题词
- 点击标签跳转到对应分类锚点
- 无雷达图区域改为信息密度更高的内容卡片区

### 缺陷 2：长列表体验差

**现状问题：** 用户滑到第 30 条，想回顶部要一直往上划，体验割裂。

**方案 A：「回到顶部」FAB 按钮**
- 位置：右下角，距边缘 16px
- 样式：圆形，白色图标（↑），毛玻璃背景
- 触发：滑动超过 1 屏后显示，否则隐藏
- 动效：淡入淡出，0.3s

**方案 B：分类锚点导航（Sticky Tab）**
- 顶部 Tab 栏 sticky 固定：全部 / AI / 科技 / 创投 / 设计
- 滑动时 Tab 吸顶，用户随时可切换分类
- 当前选中分类高亮（品牌色）
- 与 FAB 配合使用，两者不冲突

**方案 C：虚拟列表（性能优化）**
- 对于列表项 50+ 的场景，使用虚拟列表避免渲染卡顿
- 只渲染可视区域 ±3 条 buffer
- 长列表用户实际感受"快"而不是"多"

**落地优先级：** A（必须） > B（必须） > C（如有技术资源）

### 缺陷 3：内容只显示标题

**现状问题：** "某大厂发布新模型"——用户无法判断内容价值，不知道和自己有什么关系。

**新卡片结构（每条）：**

```
┌──────────────────────────────────────────┐
│ [AI]          🔥 12.3万讨论    来源：机器之心 │
│ 大模型之战：GPT-5凌晨发布，谁在焦虑？          │
│ 国内某大厂紧急召开闭门会，国产模型能否跟上…    │
│                                          │
│                    [展开] [看原文 →]       │
└──────────────────────────────────────────┘
```

**字段说明：**
- `[AI]`：分类标签（1个）
- `🔥 12.3万讨论`：热度值（优先显示讨论量，没有则显示阅读量）
- `来源：机器之心`：来源媒体（截取前15字）
- **标题**：加粗，18sp，最多2行
- **摘要**：正文前80字，灰色，14sp，2行截断
- `[展开]` 按钮：卡片内展开全文，不跳转页面
- `[看原文 →]`：跳转外部链接（仅在链接有效时显示）

**布局改版：** 卡片式替代纯列表
- 卡片间距 12px，左右各留 16px 边距
- 卡片背景白色 + 微阴影（iOS 风格）
- 热度过高（>50万）加🔥角标

### 缺陷 4：广告/评论混入内容列表

**问题本质：** 数据源头没有过滤层。

**过滤规则（三层防线）：**

**Layer 1：关键词黑名单**
```
屏蔽词库（持续维护）：
- 评论类：["网友热评", "你怎么看", "评论区", "精选评论", "粉丝留言"]
- 广告类：["限时优惠", "扫码领取", "点击下方", "立即购买", "广告"]
- 低质量：["刚刚", "突发", "震惊", "内幕曝光", "你绝对不知道"]
```

**Layer 2：规则过滤**
| 规则 | 阈值 | 处理 |
|------|------|------|
| 正文字数 | < 50 字 | 直接过滤 |
| 标题重复率 | 与历史100条中>60%相似 | 降权/过滤 |
| 来源白名单 | 非白名单来源 | 标记为"待审" |
| 链接有效性 | HTTP 状态码 ≠ 200 | 卡片内隐藏"看原文"按钮 |

**Layer 3：人工复核**
- 每日随机抽查 20 条，人工标记质量分数
- 低于阈值自动进入黑名单学习集

**落地形式：** Python 过滤器脚本，data-pipeline 层实现，0 侵入业务代码。

### 缺陷 5：点击原文显示"无内容"

**问题本质：** 详情页 data-loader 只拿到了标题字段，没有正文和链接。

**方案 A（推荐）：去掉详情页，改为卡片内展开**
- 用户点击 `[展开]` → 卡片下方展开完整正文（懒加载）
- 不需要单独的详情页，减少页面跳转
- 展开后正文最多显示 500 字，超出截断并显示"阅读更多"

**方案 B：补全详情页内容**
- data-loader 改造：同时拉取 `title / summary / content / url / hot_score / source`
- 详情页 UI 重新设计：顶部标题 + 来源 + 热度，中部正文，底部原文链接
- 技术成本：后端需要改写爬虫解析逻辑，前端需要新增详情页模板

**落地建议：** 优先方案 A，MVP 成本最低，用户体验更流畅。方案 B 作为下个迭代计划。

### 缺陷 6：毒舌一句话很尬

**重新定义 Lucia 的人设**

**核心原则：毒舌是手段，不是目的。** 毒舌的目的是让人停下来多看一秒，而不是让人尴尬到想关掉。

**人设：嘴欠但懂行**
- 知道今天发生了什么，能一针见血
- 说话像一个有态度的朋友，不是打分机器人
- 偶尔毒，但说完有信息增量

**三种毒舌模式（随机或按日切换）：**

| 模式 | 风格 | 示例（15-25字） | 使用场景 |
|------|------|----------------|---------|
| **犀利点评** | 直接说问题所在 | "ChatGPT又升级了，你的简历更新了吗？" | 有重大事件时 |
| **冷幽默** | 反讽式，绕一圈说重点 | "又一革命性突破，这次先别激动，看看谁发的。" | 营销噪音多时 |
| **毒舌+温柔** | 毒完给一句实在建议 | "模型又卷疯了，但你今天真正要看的就这一篇。" | 日报阅读氛围较轻松时 |

**字数控制：** 15-25 字，超出截断，不足不硬凑。

**生成方式（AI 辅助）：**
```
输入：今日热点话题TOP3 + 内容摘要
要求：生成一句毒舌，风格符合当日情绪基调
输出：毒舌文案
```

**位置调整：**
- 从"底部固定"改为"每10条内容后插入1条"
- 让毒舌成为阅读节奏的调剂，而不是读完后的被迫聆听

## 五、总结

这次重构的核心逻辑：**让用户知道今天发生了什么，让毒舌成为调味剂而不是主角，让脏数据在进列表之前就被过滤掉。**

情绪雷达图删掉，换成用户真正看得懂的热词标签。
列表体验做好基本功：FAB + 锚点 + 卡片信息密度。
毒舌重新做人：不尬，有信息量，位置随机而不是强行插在底部。

**第一版（MVP）做这几件事就够了：**
1. 关键词过滤脚本上线
2. 卡片增加摘要 + 来源 + 热度字段
3. FAB + 锚点 Tab
4. 热点话题标签替代雷达图
5. 毒舌重新生成，调整位置

其余可以作为后续迭代。
