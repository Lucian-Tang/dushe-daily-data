## 1. 产品设计

### 1.1 目标用户画像

| 用户群 | 特征 | 核心需求 |
|--------|------|----------|
| **AI/科技从业者** | 程序员、产品、VC | 信息密度高，没时间刷，信息质量高+有观点 |
| **独立开发者 / 技术博主** | 关注 GitHub/HN/V2EX | 快速 get 今日技术热点，需要有态度的内容 |
| **泛科技爱好者** | 大厂员工、学生、科技粉 | 把科技新闻当谈资，需要"我懂我秀"的社交货币 |
| **轻度围观群众** | 随便看看 | 被毒舌标题吸引，看个乐子 |

**主力用户：25-35 岁，科技圈 / 互联网人，有信息焦虑但时间有限。**

### 1.2 核心差异化价值

> **"不是新闻搬运工，是会吐槽的新闻搭子"**

- 竞品（微博/头条）：信息多但无观点，看完就忘
- 竞品（得到）：太重太贵，有距离感
- **我们的差异**：每条新闻都有 Lucia 的毒舌点评，给你观点而不只是信息。看完能转发、能喷、能装。

### 1.3 产品名建议

1. **毒舌日报** — 直接，简单，记忆成本低
2. **嘴替日报** — 替用户说话，"这说的就是我"
3. **Lucia 今日毒舌** — 强人格化，但偏长
4. **毒舌简报** — 偏工具感，适合商务人群
5. **每日一嘴** — 轻松，有梗，但品牌感弱

**推荐：毒舌日报**（备选：嘴替日报）

## 2. 功能 PRD

### 2.1 首页布局（1-2屏）

```
┌─────────────────────────────────┐
│  🔥 毒舌日报        [日期] [分享]│  ← 顶部 nav
├─────────────────────────────────┤
│                                 │
│  [热门毒舌卡片 1]               │  ← 大卡片，Lucia 头像 + 毒舌标题
│  "XX 公司又裁员了，这波我给负分"  │
│  🟢 AI · 🔥 999 看过            │
│                                 │
├─────────────────────────────────┤
│  [普通卡片 2]  ← 行业日报        │
│  "机器人又开始抢工作了..."       │
│  🟢 机器人 · 3分钟前            │
├─────────────────────────────────┤
│  [普通卡片 3]  ← 开发者日报      │
│  "GitHub 热榜第一：核弹级项目"   │
│  🔵 开发者 · 10分钟前           │
├─────────────────────────────────┤
│  [普通卡片 4]  ← 社会新闻        │
│  "国际大事？吃瓜时间到"          │
│  🟠 社会 · 30分钟前             │
├─────────────────────────────────┤
│  [卡片 5...]                     │
│                                 │
└─────────────────────────────────┘
```

**设计原则：**
- 第一条热门新闻占大卡片（Banner），后面是小卡片流
- 每张卡片包含：Lucia 毒舌金句 + 新闻来源标签 + 阅读量 + 时间
- 颜色标签区分内容类型：🟢 AI/行业 🟡 机器人 🔵 开发者 🟠 社会
- 底部 Tab：今日日报 / 往期 / 我的

### 2.2 交互流程

```
用户打开小程序
     ↓
首页浏览新闻列表（Lucia 毒舌预览）
     ↓
点击任意卡片 → 进入详情页
     ↓
详情页：新闻摘要 + Lucia 毒舌全文 + 相关新闻
     ↓
[毒回去] — 用户可以发表自己的毒舌评论
[分享给好友] / [分享到朋友圈]
     ↓
好友看到分享卡片 → 被毒舌吸引 → 跳转小程序
```

**关键路径：分享裂变**
- 分享卡片上要有 Lucia 毒语音金句（用户会为了这个分享而点进来）
- 朋友圈分享文案自动生成："🔥 Lucia 又在喷了：[毒舌金句]"

### 2.3 MVP 功能清单

| 优先级 | 功能 | 说明 |
|--------|------|------|
| **P0** | 新闻列表页 | 卡片流，按时间倒序 |
| **P0** | 新闻详情页 | 摘要 + 毒舌点评全文 |
| **P0** | 分享功能 | 微信小程序分享（好友/朋友圈） |
| **P0** | 内容来源标记 | AI日报/开发者日报/社会新闻标签 |
| **P1** | 毒舌评论互动 | 用户可以"毒回去"，展示评论 |
| **P1** | 往期日报 | 查看历史每天的日报 |
| **P1** | 推送通知 | 每天早 9 点推送当天日报 |
| **P2** | 个人收藏 | 收藏毒舌新闻 |
| **P2** | 热门毒舌排行 | 本周最热毒舌 TOP10 |

### 2.4 分享裂变设计

**核心策略：让用户因为"毒舌"而分享，不是因为"有用"而分享。**

| 场景 | 分享内容 | 传播动机 |
|------|----------|----------|
| 朋友圈 | Lucia 毒舌金句 + 新闻标题 | "哈哈哈哈我也想喷" |
| 好友 | 新闻卡片 + 一句话毒舌 | "这个你得看看，笑死我了" |
| 分享图片 | 毒舌语录截图 + 小程序码 | 二次传播 |

**钩子设计：**
- 每周评选"本周最毒舌"（用户评论），上榜者可获得小程序内勋章
- 分享后解锁"隐藏毒舌版"（普通版 vs 深度毒舌版内容不同）

## 3. 内容策略

### 3.1 Lucia 人格风格指南

**一句话定义：嘴贱但有温度，讽刺但不恶毒。**

```
Lucia = 毒舌闺蜜 + 技术博主 + 深夜电台主持人
```

**Do：**
- 🔥 用第一人称吐槽，"我看不下去了""这波操作我给负分"
- 😂 带梗，引用网络流行语但不过时
- 🤝 有共鸣，"打工人的崩溃就在一瞬间"
- 📊 说人话，把复杂技术翻译成人话

**Don't：**
- ❌ 不人身攻击，不针对具体人长相/隐私
- ❌ 不阴阳怪气到让人不适
- ❌ 不用太严肃的语气，"据报道"腔调
- ❌ 不蹭政治敏感话题

**毒舌程度分级：**

| 内容类型 | 毒舌程度 | 示例 |
|----------|----------|------|
| AI/科技行业 | ⭐⭐⭐ | "XX 发布新模型，参数又刷高了，有人用吗？" |
| 开发者/开源 | ⭐⭐ | "这个 GitHub 项目 star 10k，文档 404，经典！" |
| 社会新闻 | ⭐⭐⭐⭐ | "某公司裁员 30%，CEO 年终奖 2 个亿，勿cue" |
| 国际大事 | ⭐⭐⭐ | "某某国又退群了，建议成立'退圈俱乐部'" |

### 3.2 每天推送策略

| 时间 | 内容 | 形式 | 数量 |
|------|------|------|------|
| **09:00** | 早间毒舌预告 | 推送一条最重磅毒舌，吸引打开 | 1 条 |
| **12:00** | 午间加餐 | 技术圈 / AI 行业热点 | 3-5 条 |
| **18:00** | 晚报 | 社会新闻 + 国际热点 | 3-5 条 |
| **21:00** | 晚间毒舌 | 当日最精彩毒舌回顾 | 1 条 |

**推送形式：**
- 服务通知（需用户授权）
- 模板消息（点击跳转小程序）
- 推送文案示例："🔥 Lucia 今日毒舌：某大厂又画饼了，这次是 AI 驱动的元宇宙咖啡机"

## 4. 竞品/参考

### 4.1 竞品分析

| 竞品 | 特点 | 不足 |
|------|------|------|
| **得到日报** | 专业、内容重、IP 感强 | 太严肃，距离感，付费墙 |
| **快知日报** | 信息聚合，覆盖广 | 无观点，无人格 |
| **毒汤日历** | 毒舌风格鲜明 | 非新闻，内容偏生活，不够信息密度 |
| **微博热搜** | 流量大，即时性强 | 无观点，质量参差 |
| **今日头条** | AI 推荐，信息多 | 同质化，无差异化 |
| **少数派** | 科技深度文章 | 重阅读，非快资讯 |

### 4.2 我们差异点

| 维度 | 竞品 | 毒舌日报 |
|------|------|----------|
| **人格** | 无 / 弱 | Lucia 毒舌 IP，有黏性 |
| **观点** | 无 | 每条新闻都有毒舌点评 |
| **速度** | 快（微博）vs 慢（得到） | 快（实时抓取）+ 深（每日汇总）|
| **传播** | 靠标题党 | 靠毒舌金句驱动的主动分享 |
| **技术背景** | 无 | 我们有三大日报自动生成系统，壁垒高 |
| **定价** | 付费（得到） | MVP 阶段免费 |

### 4.3 参考产品灵感

- **即刻** — 垂直兴趣圈，高粘性社区
- **毒汤日历** — 毒舌内容调性
- **Projector / Stoned** — 内容消费新形态，故事性强

## 5. MVP 里程碑（2-3天）

```
Day 1：产品设计 + 设计稿（Thomas）
Day 2：前端开发（小程序框架）— 小布
Day 3：后端对接 + 内容填充 + 上架审核
```

**MVP 验收标准：**
- ✅ 用户能看到三条日报内容（AI/开发/社会）
- ✅ 每条新闻有 Lucia 毒舌点评
- ✅ 可以分享到微信好友/朋友圈
- ✅ 小程序流畅，无明显卡顿
