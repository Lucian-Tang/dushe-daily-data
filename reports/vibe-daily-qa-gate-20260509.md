# 🛡️ Vibe-Daily v2.1 QA 质量门禁报告

**审核人:** QA 校验官  
**审核日期:** 2026-05-09  
**版本:** v2.1  

---

## 判定：🟡 条件通过

---

## 理由

1. **P0 代码缺陷未全部修复（阻塞）**  
   - B1（isToday 时区问题）、B3（onScroll 无 throttle）、R3（detail_cache 覆盖）三项 P0 问题在代码层面尚未修复，今日计数/低端机性能/详情页数据错乱风险依然存在，不适合直接发布。

2. **UI 存在明确 P0 差距，影响核心体验**  
   - 详情页缺少固定底部操作栏（收藏/分享），用户看完需滚回顶部操作；正文字号 15px 偏小，可读性不如竞品 16-17px。两项均属 P0 优先级，建议上线前修复。

3. **P1 问题有部分排期或理由，可接受发布**  
   - UI 报告中的 P1 项（主题切换、小红书海报）有明确优化方向；代码中 R6（Canvas 超时）虽有风险但仅影响低端机用户子集。可在 v2.2 迭代中修复。

---

## 阻塞项

| 优先级 | 问题 | 说明 | 修复方式 |
|--------|------|------|----------|
| **P0** | B1 isToday() 时区问题 | 跨时区数据凌晨被误判为"昨天"，今日计数不准确 | 统一用 `toISOString().slice(0,10)` 字符串比较 |
| **P0** | B3 onScroll 高频 setData | 低端机滚动严重卡顿 | 加 200ms throttle |
| **P0** | R3 detail_cache 单条覆盖 | 快速点击两张卡片会互相覆盖 | 改为 `detail_cache_{id}` Map 存储 |
| **P0** | 详情页缺少固定底部操作栏 | 收藏/分享需滚回顶部操作，转化率损失 | 固定底部操作栏（收藏+分享+👍👎） |
| **P0** | 详情页正文字号 15px 偏小 | 可读性差于竞品 16-17px 标准 | 调整为 16px，1.8 行高 |

---

## 非阻塞 P1（建议 v2.2 排期）

| 问题 | 描述 |
|------|------|
| B2 getRandomLuciaComment 空数组 | 特殊场景下毒舌 quote 缺失，已有兜底逻辑，可延后 |
| R2 calcHotScore 随机性 | 热度值每次不同，同一文章切换 tab 分数波动，建议 hash seed |
| R6 Canvas 无超时保护 | 低端机海报生成可能卡死，建议加状态标记+失败重试 |
| UI P1：主题切换 | 日间/夜间跟随系统主题，户外可读性优化 |
| UI P1：浅色小红书海报 | 浅底+品牌色风格，提升该平台传播效果 |

---

## 总结

v2.1 发布前需完成 **5 项 P0 修复**（3 项代码 + 2 项 UI），可在 1-2 天内完成。P1 问题可在 v2.2 迭代处理，不阻塞当前发布。

**建议：修复 5 项 P0 后再发布，或明确告知用户当前为 Beta 版本。**
