# 琢光旗舰产品重构方案

> 产品：琢光（Luxmind P0 旗舰产品）
> 阶段：从「品味挑战小程序」升级为「AI时代品味阶层平台」
> 作者：Lucia
> 日期：2026-05-09
> 状态：战略方案 v1.0

---

## 一、产品愿景

**琢光 = AI时代品味阶层的养成平台。**

三个核心动词：**培养 → 判断 → 展示**

不是做题工具，不是内容App，不是人格测试。是**一个用户在AI时代建立自己审美/判断/表达身份的完整系统**。

---

## 二、竞品全景扫描

### 2.1 按三个维度拆解竞品

#### 培养（内容鉴赏+反馈）

| 竞品 | 形态 | 做什么 | 缺什么 |
|------|------|--------|--------|
| **DailyArt** | App | 每天推一幅经典艺术作品+解读 | 纯单向，无互动反馈，无数据积累 |
| **Google Arts & Culture** | App/Web | 全球博物馆藏品浏览 | 太宽泛，无个性化推荐，无训练体系 |
| **小红书** | App | UGC审美内容社区 | 算法推荐而非训练，无判断力评估 |
| **Artsy** | Web | 艺术品交易+鉴赏 | 面向收藏者，非普通人 |
| **Vimeo/Nowness** | Web | 精选短片+影像艺术 | 纯内容消费，无互动机制 |

**空白：** 无人做「精选内容→用户反馈→数据积累→人格画像」的闭环。

#### 判断（答题+语音表达）

| 竞品 | 形态 | 做什么 | 缺什么 |
|------|------|--------|--------|
| **Duolingo** | App | 游戏化语言学习 | 品类不同，但游戏化机制可借鉴 |
| **Brilliant** | App/Web | 交互式数学+科学学习 | 品类不同，互动设计一流 |
| **Elevate** | App | 大脑训练游戏 | 偏认知能力（记忆力/计算），非审美判断 |
| **PromptBase** | 市场 | 提示词交易 | 只卖不教，无训练体系 |
| **AI Prompt 挑战赛** | Web | 社区Prompt比赛 | 一次性活动，无持续训练 |
| **Clubhouse** | App | 语音社交 | 无表达训练，无评估反馈 |
| **CapCut/剪映** | App | 视频创作 | 创作工具，非训练平台 |

**空白：** 无人做「审美判断力+提示词表达力+语音即兴力」的系统训练。Duolingo的机制（段位/连胜/排行榜/每日打卡）可以直接参考，但品类完全不同。

#### 展示（AI身份名片）

| 竞品 | 形态 | 做什么 | 缺什么 |
|------|------|--------|--------|
| **16Personalities** | Web | MBTI测试+人格描述 | 一次性的，不与行为数据联动 |
| **Spotify Wrapped** | App内功能 | 年度听歌报告 | 数据驱动的身份展示，但一年一次 |
| **人格测试类小程序** | 小程序 | 各种测试+分享卡片 | 娱乐性，无真实数据支撑 |
| **Character.AI** | App | AI角色创建 | 创造虚拟人格，非展示真人品味 |
| **Soul App** | App | 灵魂社交 | 兴趣匹配，非品味身份 |
| **即刻** | App | 兴趣社区 | 有圈子标签但无量化评估 |

**空白：** 无人用**真实行为数据**（而非自填问卷）生成人格名片。这是核心护城河。

### 2.2 竞品总结

**没有直接竞品。** 没有任何产品同时做「鉴赏培养+判断训练+身份展示」三位一体。每个维度都有分散的竞品，但合在一起是空白。

**最值得借鉴的机制：**
- Duolingo：段位体系、每日打卡、排行榜、连胜激励
- Spotify Wrapped：数据驱动身份展示、年度报告
- Brilliant：交互式学习体验
- Elevate：游戏化能力训练

## 三、产品机制设计

### 3.1 核心循环

```
用户进入  →  鉴赏一段精选内容（今日艺术品/设计/影像）
    ↓
用户反馈（A/B选择 / 语音评论 / 看图描绘）
    ↓
AI实时评估 + 数据积累（品味分/判断力/表达力）
    ↓
展示身份变化（段位升级 / 人格名片更新 / 今日洞察）
    ↓
分享裂变 → 新用户进入
```

### 3.2 培养模块：品味鉴赏

**每日精选内容类型：**
- 经典艺术（DailyArt模式）：一幅画/雕塑+背景故事
- 设计对比（现有题型）：两个设计，哪个更好？为什么？
- AI vs 人类（现有题型）：判断哪个是AI生成，训练AI辨别力
- 影像片段：30秒短片/摄影作品，感受+评论
- 文字段落：经典文学/广告文案/品牌Slogan评比

**互动机制：**
- 看完内容后，不只做选择题
- 滑动手势表达「喜欢/无感/有意思」（类似Tinder，数据采集更自然）
- 可选：15秒语音评论（"看完这幅画你想说什么"）
- AI 实时转写+情绪分析+内容评估

**数据积累维度：**
- 审美偏好向量（喜欢什么类型的艺术/设计/文字风格）
- 感知敏锐度（能否注意到细节）
- 表达能力（语音评论的用词丰富度、逻辑清晰度）

### 3.3 判断模块：能力训练

**三大题型体系：**

**题型一：二选一判断（现有，保留+强化）**
- AI vs 人类识别
- 设计A vs B
- 文案对比评判
- 提示词A vs B（哪个prompt效果更好？）

**题型二：看图描绘（新增）**
- 展示一张图，用户用文字描述/或语音描述
- AI评估：观察力（描述了多少细节）、表达力（用词精准度）、创造力（独特视角）
- 这直接对接「卖课」——表达力不够？来学提示词工程

**题型三：即兴语音（新增）**
- 给一句话/一个场景，30秒语音即兴回应
- 如：「用一句话说服我为什么这张图比那张好」
- AI评估：逻辑、感染力、用词、语速节奏
- Z世代天然喜欢语音交互

### 3.4 展示模块：AI身份名片

**身份数据来源（全部基于真实行为，非自填）：**

| 维度 | 数据来源 | 展示 |
|------|----------|------|
| 品味段位 | 鉴赏+判断正确率 | 👁️观察者→🗼灯塔 |
| 审美偏好 | 内容浏览+滑动记录 | "你的审美DNA：极简主义·冷色调·抽象" |
| 判断力 | 答题正确率+决策速度 | "判断力 85/100" |
| 表达力 | 语音/文字评论分析 | "表达风格：犀利简洁型" |
| 提示词力 | Prompt对比题正确率 | "提示词段位：银" |
| Vibe标签 | AI综合分析 | "理性·毒舌·古典品味" |

**名片展示形态：**
- 可分享的图片卡片（类似Spotify Wrapped）
- 动态二维码（扫码看到最新版本）
- 名片历史时间线（品味进化史）

## 四、护城河分析

### 4.1 护城河层级

**第一层：行为数据护城河（最深）**

品味分数不是你自己选的，是千百次选择累积的。竞品可以抄UI，抄不了「用户的真实判断数据」。越用越准，迁移成本越高。

**第二层：三位一体生态护城河**

培养→判断→展示不是三个功能，是一个闭环。单做任何一个维度的竞品，都无法形成数据闭环。用户只用鉴赏功能的人，看到别人有名片，就想做题。只做题的人，看到别人有鉴赏内容，就想看。互相导流。

**第三层：内容+题库护城河**

精选鉴赏内容需要人工策展+AI推荐。判断题库需要持续更新（AI生成内容变化太快）。光靠UGC不够，需要专业编辑能力。这是一个内容运营壁垒。

**第四层：品牌护城河**

Luxmind = AI时代品味身份。这个品牌认知一旦建立，所有后来者都是在做"模仿Luxmind的东西"。

### 4.2 可防御性评估

- 微信小程序复制：容易（UI可抄），但数据抄不了
- 大厂入场：可能（字节/腾讯可能做类似功能），但大厂不会专注做"品味身份"这个小切口
- 创业公司：最容易的抄袭是单功能（只做人格名片），但无法形成三位一体闭环
- **终极护城河：用户在你这里的品味分数/段位，换个平台就归零**

## 五、产品架构重构

### 5.1 从「单一答题小程序」到「品味养成平台」

**当前架构（琢光 V0.4.x）：**
```
首页 → 选择模式 → 答题 → 结果 → 个人
（Tab：挑战/练习/排行/我的）
```

**目标架构（琢光 V2.0）：**
```
┌────────────────────────────────────────────┐
│                  琢光                        │
├────────────────────────────────────────────┤
│                                            │
│  今日（首页）                               │
│  ├─ 每日精选鉴赏内容（图文/视频/音频）       │
│  ├─ 今日挑战（3道判断题）                   │
│  └─ 昨日回顾（你的品味分数变化）             │
│                                            │
│  训练（Tab 2）                              │
│  ├─ 判断训练（分类：AI识别/设计/文案/提示词） │
│  ├─ 表达训练（看图描绘/即兴语音）            │
│  └─ 专项突破（弱项强化）                    │
│                                            │
│  鉴赏（Tab 3）                              │
│  ├─ 每日精选流（滑动浏览+反馈）              │
│  ├─ 分类浏览（艺术/设计/影像/文字）          │
│  └─ 语音评论入口                            │
│                                            │
│  名片（Tab 4）                              │
│  ├─ 我的AI身份名片                           │
│  ├─ 品味进化史                               │
│  ├─ 分享/扫码                                │
│  └─ 好友契合度                               │
│                                            │
│  我的（Tab 5）                              │
│  ├─ 段位+数据看板                            │
│  ├─ 成就墙                                  │
│  └─ 设置                                    │
│                                            │
└────────────────────────────────────────────┘
```

### 5.2 核心改动

1. **首页从「开始挑战」变成「今日精选+挑战」** — 先鉴赏再判断，培养和判断一体
2. **新增「鉴赏」Tab** — 替代原来的「练习」Tab，变成内容浏览+反馈入口
3. **「排行」Tab改为「名片」Tab** — 排行榜功能弱化，身份展示功能强化
4. **语音交互贯穿全局** — 鉴赏内容可语音评论，判断训练有语音即兴题型

### 5.3 商业化落点

**免费层：**
- 每日鉴赏内容（3条/天）
- 每日挑战（5题/天）
- 基础名片（3维度）
- 基础段位

**Pro层（¥38/月 或 ¥338/年）：**
- 无限鉴赏内容 + 历史回顾
- 无限训练题库 + 专项突破
- 全维度名片 + 名片历史
- 表达力训练（看图描绘+即兴语音）+ AI评分反馈
- 品味进化报告（月度/年度）

**教育变现（未来）：**
- 提示词工程课程（表达力不够→买课）
- 审美能力提升课（品味分低→买课）
- 语音表达训练营

## 六、开发落地路径

### Phase 1：基础重构（Week 1-2）

**目标：** 现有答题功能打磨到可上线状态 + 首页改版

- [ ] UI重构：新的5Tab结构（今日/训练/鉴赏/名片/我的）
- [ ] 首页改版：今日精选鉴赏内容（静态内容，先人工策展5条）
- [ ] 答题功能完善（现有题型Bug修复+题库扩充到50题）
- [ ] 基础名片生成（基于答题数据，3维度）
- [ ] 段位体系接入名片

**上线标准：** 用户可用答题+鉴赏+看名片

### Phase 2：鉴赏内容系统（Week 3-4）

**目标：** 鉴赏Tab完整上线 + 数据采集

- [ ] 鉴赏内容管理后台（人工策展→AI推荐）
- [ ] 内容浏览+滑动反馈机制
- [ ] 语音评论功能（录制→AI转写→情绪分析→数据积累）
- [ ] 鉴赏数据接入身份评估模型

**上线标准：** 用户可浏览每日精选、滑动反馈、语音评论

### Phase 3：表达训练+语音互动（Week 5-7）

**目标：** 判断模块从纯选择题扩展到语音题型

- [ ] 看图描绘题型（30秒语音描述图片→AI评分）
- [ ] 即兴语音题型（给定场景→30秒即兴回应→AI评分）
- [ ] 表达力评估模型（用词/逻辑/感染力/语速）
- [ ] 表达力数据接入名片

**上线标准：** 用户可以完成语音题型并获得评分

### Phase 4：社交裂变+身份体系（Week 8-10）

**目标：** 名片社交化 + 增长飞轮启动

- [ ] 全维度AI身份名片（5维度）
- [ ] 名片分享+扫码互看
- [ ] 好友契合度
- [ ] 品味进化时间线
- [ ] Pro订阅体系上线

**上线标准：** 用户可分享名片、互看契合度、开通Pro

### Phase 5：AI推荐+个性化（Week 11+）

- [ ] 鉴赏内容AI推荐（基于用户审美DNA）
- [ ] 弱项识别+专项突破推荐
- [ ] 品味进化年度报告（mini Spotify Wrapped）
- [ ] 教育变现入口（提示词课/审美课）

## 七、关键风险与应对

| 风险 | 可能性 | 影响 | 应对 |
|------|--------|------|------|
| 语音交互体验差（ASR不准/AI评分不合理） | 中 | 高 | Phase 2先灰度，核心经验证后再推广 |
| 鉴赏内容版权问题 | 中 | 中 | 优先使用公共领域+CC协议内容，后续合作美术馆 |
| 题库枯竭（AI生成内容变化快，旧题过时） | 中 | 中 | 设计自动化题库生成管线（AI新模型发布→自动生成对比题） |
| 用户留存低（鉴赏+答题不够上瘾） | 高 | 高 | 强化游戏化（连胜/段位/排行榜），语音互动增强粘性 |
| 微信小程序审核风险（语音/内容） | 低 | 高 | 鉴赏内容严格审核，语音做本地处理不过服务器 |
| 大厂复制 | 中 | 高 | 加速跑马圈地，行为数据护城河越深越好 |

---

## 八、总结

**琢光不是一个小程序改版，是一次产品哲学升级。**

从「做题工具」升维到「品味养成平台」。
从「单一功能」升维到「培养→判断→展示」三位一体。
从「用完就走」升维到「每天来打卡的身份投资」。

**12周路线图：** 基础重构→鉴赏内容→语音训练→社交裂变→AI推荐。

**核心护城河：** 真实行为数据驱动的品味身份。竞品抄得走UI，抄不走你的琢光分数。
