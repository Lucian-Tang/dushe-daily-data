> 产品经理：Thomas

## 一、V2.0 产品目标（3个月目标）

**核心使命：** 在不展示原文的前提下，用最低认知成本让用户知道「这条新闻值不值得信」。

**三个月目标（2026年5月 - 8月）：**

| 维度 | 目标 |
|------|------|
| 信任感知 | 可信度评分渗透率 >60%，用户认可评分「说得通」 |
| 留存 | 次日留存率 +5pp（基准对比 V1.5） |
| 分享 | 分享率 +20%，「评分截图」成为社交货币 |
| 商业 | 付费频道付费率稳中有升，不因评分降低付费意愿 |

**不做什么：**
- 不做新闻全文展示
- 不做投资建议类评分
- 不做多语言/境外内容

## 二、功能范围（Phase 1 最小集）

### Phase 1 最小可行功能（MVP）

**2.1 可信度评分系统**

- **5星评分体系**：每条新闻卡片附可信度星级（1-5星）
- **评分维度**（对用户透明）：
  - 信源历史准确率（官方媒体 > 自媒体）
  - 交叉验证数量（多少独立信源报道同一事件）
  - 事件时间距离（突发事件 vs 旧闻翻炒）
- **评分展示**：星级 + 简短关键词（如「多信源确认」「单一来源」）——不展示原始数据细节
- **合规标注**：每张卡片评分旁显示「AI生成，仅供参考」

**2.2 信源透明度卡片**

- 点击评分区域 → 弹出轻量信源说明（信源名称 + 为什么给这个分）
- 不展示原始新闻链接（合规）
- 不展示新闻摘要全文（合规）

**2.3 毒舌点评升级**

- 在现有毒舌点评末尾增加「可信度快评」一句（如：「这消息 Three directions 三路人马都报了，可信度高，但别高兴太早——上次这么热闹的事最后反转了。」）
- AI生成标注：「AI点评，仅供参考」

## 三、卡片UI改造方案

### 设计原则

> **「不增加认知负担」= 用户在0.5秒内能感知「这条信不信」**

### 改造方案

**现有卡片结构（V1.5）：**
```
[频道标签] 标题 毒舌点评 [收藏] [分享]
```

**V2.0 卡片结构：**
```
[频道标签] [可信度⭐⭐⭐] 标题
第一行毒舌点评
[信源说明文字 · AI生成，仅供参考]
[收藏] [分享海报]
```

**评分展示规则：**

| 星级 | 视觉呈现 | 颜色 | 含义 |
|------|---------|------|------|
| ⭐⭐⭐⭐⭐ | 5星金色 | 暖黄 | 多方印证，高度可信 |
| ⭐⭐⭐⭐ | 4星 | 浅金 | 基本可信，少量独立来源 |
| ⭐⭐⭐ | 3星 | 灰蓝 | 中性，单一来源或事件较新难判断 |
| ⭐⭐ | 2星 | 橙色 | 存疑，信息源可疑或逻辑存疑 |
| ⭐ | 1星 | 红色 | 高风险，虚假/高度疑似传言 |

**降低认知负担的具体设计：**

1. **星级默认折叠展示**：用户不主动点击不展开详细维度
2. **颜色编码 > 数字**：红色就是危险，黄色就是可信，一眼看懂
3. **点击展开「为什么给这个分」**：hover/tap 显示 2-3 行解释文字，不展示原始数据
4. **评分与毒舌点评分离**：评分是冷静的「仪表盘」，毒舌是情感化的「吐槽」，两者不混淆

**「这个评分怎么来的」透明度入口：**

```
点击星级区域 → 弹出浮层（不离开卡片）：
┌─────────────────────────┐
│  可信度分析 · AI生成    │
│  ─────────────────────  │
│  ✓ 新华社、人民日报、   │
│    央视新闻均有报道     │
│  ✓ 事件发生于2小时内    │
│  ✓ 3家以上媒体独立核实  │
│  ─────────────────────  │
│  [收起]                 │
└─────────────────────────┘
```

## 四、用户体验升级

### 4.1 评分可视化

- **首页瀑布流**：高可信度新闻（4-5星）卡片左上角加「稳」角标；低可信度（1-2星）加「⚠️」警示
- **筛选能力**：未来可增加「只看可信新闻」开关（Phase 2）
- **数据可视化**：个人中心展示「本周阅读新闻可信度分布」雷达图（Phase 2）

### 4.2 信源透明度

- **信源白名单机制**：建立内部信源分级库（官方媒体 / 优质自媒体 / 普通自媒体 / 可疑信源）
- **信源卡片**：长期活跃信源显示「来源：XXX（已验证）」标识
- **用户教育**：首次打开评分功能时，显示 3 张引导卡，说明评分含义（看过即消失，不重复打扰）

### 4.3 毒舌升级

- **可信度联动毒舌**：毒舌文案风格随评分变化
  - 高可信（4-5星）：毒舌更自信，调侃空间大
  - 低可信（1-2星）：毒舌更刻薄，警告语气更强
- **新增「可信度一句」**：每条新闻毒舌点评末尾追加 1 句可信度简评
- **合规双重标注**：「毒舌点评」区域和「可信度评分」区域均标注「AI生成，仅供参考」

### 4.4 分享体验

- **分享海报升级**：分享卡片自动附带星级，用户截图分享时自带可信度背书
- **「可信截图」功能**：一键生成带评分和信源说明的图片，适合发朋友圈/群（Phase 2）

## 七、风险与对策

| 风险 | 概率 | 影响 | 对策 |
|------|------|------|------|
| 评分算法误判导致用户投诉 | 中 | 高 | Phase 1 灰度上线，算法可人工干预 |
| 用户误解评分为投资建议 | 低 | 高 | 评分体系明确声明「不含投资判断」，UI层面隔离 |
| 合规检查发现AI标注遗漏 | 中 | 高 | 自动化扫描工具，每次发版前检查 |
| 付费用户认为评分影响内容价值 | 低 | 中 | 付费频道评分功能独立设计，不降低优质内容评级 |
