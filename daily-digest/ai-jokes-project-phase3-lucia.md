## 一、每日文案模板设计 ✅

### 糗事文案结构模板

```
【事件简述】
[一句话描述AI/机器人做了什么糗事，30字以内]

【吐槽角度】
[从旁观者/受害者角度的犀利点评，50字以内，要有梗感]

【互动引导】
[引导评论/转发的互动问题，15字以内，要有趣]
```

### 分类prompt

**AI Fails类（明确翻车）：**
```
你是一个社交媒体内容策划，擅长写"AI糗事"类段子。

任务：根据提供的AI失败案例，写一条微博风格的吐槽文案。

要求：
1. 事件简述：客观描述发生了什么，30字以内
2. 吐槽角度：犀利但不失幽默，50字以内，要让人想评论
3. 互动引导：一句有趣的问题，引导互动，15字以内

输出格式：
【事件简述】xxx
【吐槽角度】xxx
【互动引导】xxx
```

**AI Mistakes类（AI犯蠢）：**
```
你是一个毒舌但可爱的AI吐槽博主，专门嘲笑AI的"犯蠢"时刻。

任务：根据以下AI错误案例，写一条轻松幽默的吐槽文案。

要求：
1. 事件简述：描述AI哪里搞错了，30字以内
2. 吐槽角度：从用户角度吐槽为什么这个错误让人崩溃/好笑，50字以内
3. 互动引导：引导大家分享类似经历，15字以内

注意：语气要毒舌但可爱，不要太严肃
```

## 二、敏感内容过滤规则

### 2.1 绝对禁止词汇（触碰即删）

| 类别 | 词汇 |
|------|------|
| 政治/敏感人物 | 习近平/国家领导人/台独/藏独/疆独等 |
| 暴力/犯罪 | 恐怖袭击/杀人/自杀方法等 |
| 色情/低俗 | 详细性描述/未成年人相关 |
| 歧视/仇恨 | 种族歧视/性别歧视/地域歧视 |
| 医疗/药物 | 毒品名称/处方药引导 |

### 2.2 高风险词汇（需人工审核）

| 类别 | 词汇/场景 |
|------|---------|
| 公司/品牌 | 明确点名大公司（Google/Meta/OpenAI等）需检查是否负面 |
| 人物 | 具体人名（非公众人物）需检查隐私 |
| 争议事件 | 自杀/猝死/事故现场等需谨慎 |
| 医疗建议 | 任何看起来像医疗建议的内容 |

### 2.3 过滤流程

```
原始数据输入
    ↓
绝对禁止词扫描 → 发现 → 直接过滤掉
    ↓ 通过
高风险词扫描 → 发现 → 标记"待人工审核"
    ↓ 通过
敏感事件检测 → 政治/暴力/色情 → 标记"待人工审核"
    ↓ 通过
进入文案生成环节
```

## 三、生成质量人工抽检机制

### 3.1 抽检比例

| 内容类型 | 抽检比例 |
|---------|---------|
| 常规糗事内容 | 100% 机器生成 → 20%人工抽检 |
| 涉及公司/品牌 | 100% 人工审核 |
| 争议性话题 | 100% 人工审核 |
| 首发内容 | 100% 人工审核 |

### 3.2 抽检标准

**通过标准（全部满足）：**
- [ ] 内容与原事件事实相符，无编造
- [ ] 吐槽角度合适，不人身攻击
- [ ] 没有敏感词（通过过滤扫描）
- [ ] 字数在规定范围内
- [ ] 有互动引导，适合发布

**不通过处理：**
- 严重问题 → 删除，重新生成
- 轻微问题 → 修改后发布
- 存疑 → 标记待Boss决策

### 3.3 抽检SOP

```
1. 获取生成内容（文案+配图）
2. 执行敏感词扫描（全量）
3. 人工阅读内容，检查：
   - 事实准确性
   - 语气是否合适
   - 是否有隐性风险
4. 通过 → 标记可发布
   不通过 → 记录问题，返工生成
5. 每日抽检记录归档
```

### 3.4 抽检记录格式

```markdown
## 抽检记录 YYYY-MM-DD

| 序号 | 内容摘要 | 抽检时间 | 结果 | 问题描述 |
|------|---------|---------|------|---------|
| 1 | [文案摘要] | HH:MM | ✅/❌ | [如有问题] |
```
