## 二、标题公式体系

### 2.1 三大公式（固定轮换）

| 公式类型 | 公式 | 示例 |
|---------|------|------|
| 悬念式 | 「X个信号表明XXX正在爆发」 | 「3个信号表明AI编程工具正在爆发」 |
| 对比式 | 「XXX vs XXX：谁赢了？」 | 「Cursor vs Copilot：谁才是真王者？」 |
| 数字清单式 | 「XX个你应该知道的XXX」 | 「7个GitHub本周最火的项目」 |

### 2.2 副标题规则
- 资源补充型：「附工具/资源/竞品对比」
- 情绪共鸣型：「打工人的效率核武器」「开发者必读」
- 禁忌词：❌ 最全/最强/必看/震惊/突发

### 2.3 标题自检清单
- [ ] 通过公式验证
- [ ] 无禁用词
- [ ] 情绪适中（不高调不标题党）
- [ ] 核心关键词前置
- [ ] 平台长度适配（公众号≤32字，X≤28字）

## 三、内容模块标准化

### 3.1 标准五模块（每期固定）

```
📰【今日要点】  — 3句摘要，≤60字，点明核心冲突/主题
🔥【热点速递】  — Top3热点，每个含：平台/热度/一句话简介
💡【深度1+1】  — 2个深度分析，每个≤150字（观点→展开）
🛠️【工具推荐】  — 1-2个工具，含简介+直达链接
📅【本周预告】  — 下周值得关注的3件事
```

### 3.2 变体模板（按场景轮换）

| 变体 | 适用场景 | 增加模块 |
|------|---------|---------|
| 数据驱动型 | 数据丰富时 | 📊「本周数据速览」 |
| 事件驱动型 | 有大事件时 | 🏆「本周大事记」 |
| 资源包型 | 有集合资源时 | 🎁「资源大礼包」 |

### 3.3 各渠道内容适配规则

**微信公众号（主渠道）**
- 完整版，所有模块保留
- 配图：封面900×383px，次图800×450px
- 可嵌入小程序/外部链接

**知乎专栏（SEO版）**
- 开头加「本文导航」便于跳转
- 标签打满5个相关标签
- 标题含核心关键词利于搜索

**X/Twitter（摘要版）**
- 主推文：标题+一句话+link，≤280字
- 准备1-2条讨论推文增加互动
- 配图：1200×675px（16:9）

**RSS/Newsletter（邮件版）**
- 邮件主题：「🔥 [日期] 科技早报 | Top3...」
- 预览文字前100字决定打开率，精心撰写

## 四、配图规范

### 4.1 规格对照表

| 平台 | 规格 | 比例 |
|------|------|------|
| 封面通用 | 1200×630px | 16:9 |
| 微信公众号封面 | 900×383px | 2.35:1 |
| 微信公众号次图 | 800×450px | 16:9 |
| 知乎封面 | 840×460px | 16:9 |
| X/Twitter卡片 | 1600×900px | 16:9 |
| RSS/OG Image | 1200×630px | 16:9 |

### 4.2 风格规范

**颜色**
- 主色调：深蓝/科技蓝 #1A73E8 或紫色 #6C5CE7
- 背景：深色 #1a1a2e 或渐变，禁纯白
- 文字：白色/亮色，在深色背景上可读

**字体**
- 标题：Arial Bold / 思源黑体 Bold
- 正文：Arial / 思源黑体 Regular
- 数字/标签：Roboto Mono

**内容要求**
- 封面图必须包含：本期内核关键词 + 日期 + 品牌标识（小）
- 禁止：纯截图、低分辨率、版权图片

### 4.3 AI配图提示词模板

**封面图：**
```

stylized data visualization elements, bold headline text placeholder,
professional and clean design, 16:9 aspect ratio, no photographs
```

**内容辅助图：**
