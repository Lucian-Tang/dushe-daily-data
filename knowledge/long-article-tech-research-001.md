## 一、长文生产标准化SOP流程

### 1.1 五阶段总览

```
热点选题 → 素材采集 → 文案撰写 → 配图生成 → 多平台发布
    ↓           ↓           ↓          ↓          ↓
  Thomas     Stephen     Thomas    Stephen    Lucia/Thomas
```

### 1.2 各阶段输入/输出/时长

| 阶段 | 输入 | 输出 | 标准时长 | 异常处理 |
|------|------|------|---------|---------|
| **热点选题** | 各平台热榜数据、行业动态 | 选题列表（含优先级） | 30min | 热度不足时切换备选选题 |
| **素材采集** | 选题关键词 | 素材包（数据/案例/链接） | 60min | 素材不足时扩大关键词范围 |
| **文案撰写** | 素材包 | 初稿（800-3000字） | 90min | 素材不足时缩减篇幅或换选题 |
| **配图生成** | 初稿核心观点 | 封面图+内容图 | 30min | 生成失败切手动配图 |
| **多平台发布** | 定稿+配图 | 各平台已发布 | 45min | 平台审核不通过时调整格式重提 |

### 1.3 热点选题标准

**选题来源优先级：**
1. Twitter/X 热帖（实时性强）
2. Hacker News Top（科技圈关注）
3. GitHub Trending（开发者生态）
4. 36氪/机器之心（行业媒体）
5. 百度/头条热榜（大众热点）

**选题评估维度：**
- 热度：相关社区讨论量 > 100
- 独家性：是否有新角度
- 可操作性：素材是否易获取
- 时效性：是否值得现在发布

**输出格式：**
```markdown
## 选题：[标题]
- 来源平台：[平台]
- 热度评估：[高/中/低]
- 核心角度：[切入方向]
- 素材关键词：[可搜索关键词]
- 预计字数：[范围]
```

## 二、多平台格式适配方案

### 2.1 微信公众号

| 项目 | 规格 |
|------|------|
| 封面图 | 900×383px（2.35:1），≤5MB，JPG/PNG |
| 次图 | 800×450px（16:9） |
| 正文段落 | 每段 3-4 行，不超过手机一屏 |
| 段落间距 | 1.75倍行高 |
| 摘要 | ≤120字，决定打开率 |
| 标题 | ≤32字，含核心关键词 |
| 链接 | 文末"阅读原文"跳转，外链需备好 |

**正文格式规范：**
- 一级标题：黑体 18px，加粗
- 二级标题：黑体 16px，加粗
- 正文：宋体 15px，1.75倍行高
- 金句：居中，16px，可加底色框
- 图片注释：居中，灰色，12px

### 2.2 知乎专栏

| 项目 | 规范 |
|------|------|
| 开头导航 | 必须有「本文目录」跳转锚点 |
| 标签 | 打满5个相关标签 |
| SEO标题 | 前60字含核心关键词 |
| 开头 | 前200字决定推荐量，要有信息增量 |
| 图片 | 840×460px（16:9） |

**SEO技巧：**
- 标题含「XXX教程/评测/对比/指南」等词
- 正文前300字多关键词自然分布
- 结尾加「相关问题」引导
- 评论区运营：发布后1小时内回复评论

### 2.3 36氪/其他科技媒体

| 项目 | 规范 |
|------|------|
| 投稿方式 | 官网投稿入口或编辑邮箱 |
| 篇幅 | 1500-2500字 |
| 格式 | 偏好Markdown，禁多级嵌套列表 |
| 配图 | 自制配图优先，禁止版权图片 |
| 审核周期 | 24-72小时 |

**36氪特有问题：**
- 偏好深度行业分析，非资讯类
- 需要明确的「编辑亮点/价值点」
- 投稿后需持续跟进编辑反馈

### 2.4 各平台差异对照表

| 项目 | 微信公众号 | 知乎专栏 | 36氪 |
|------|-----------|---------|------|
| 最佳发布时段 | 08:00-09:00 | 14:00-16:00 | 工作日白天 |
| 图片比例 | 2.35:1封面 | 16:9 | 16:9 |
| 标题字数 | ≤32字 | ≤40字 | ≤36字 |
| SEO需求 | 低 | 高 | 中 |
| 评论运营 | 弱 | 强 | 中 |
| 审核机制 | 无（先发后审） | 无 | 有（先审后发） |

## 四、标准化长文生成SOP文档

### 4.1 完整执行流程

**Step 1：热点选题（30min）**
```
1.1 拉取各平台热榜（cn-trends-aggregator）
1.2 筛选热度>100 或 有独家角度的选题
1.3 评估素材可获取性
1.4 确定最终选题（1个主选 + 2个备选）
输出：选题清单
```

**Step 2：素材采集（60min）**
```
2.1 搜索选题相关数据点（Google/行业报告）
2.2 收集2-3个具体案例（含人名/地点）
2.3 整理权威引用来源
2.4 整理素材包：[数据]+[案例]+[引用]+[链接]
输出：素材包
```

**Step 3：文案撰写（90min）**
```
3.1 生成10个标题候选
3.2 选定最优标题
3.3 按模板套写初稿
3.4 人工润色（检查逻辑/节奏/数据准确）
输出：定稿
```

**Step 4：配图生成（30min）**
```
4.1 生成封面图（主题关键词 + 日期）
4.2 生成1-2张内容辅助图
4.3 按平台规格裁剪
输出：配图包
```

**Step 5：多平台发布（45min）**
```
5.1 微信公众号发布（基准平台）
5.2 知乎专栏改写格式 + 发布
5.3 36氪投稿（如适用）
5.4 X/Twitter摘要 + 发布
5.5 数据回收（阅读/评论/转发）
输出：已发布各平台
```

## 六、附录：长文模板

### A. 科技爆款模板（1800-3000字）

```
【开篇钩子】（150字内）
反常识事实/震撼数据/具体场景

【Section 1：事件本身】（400-600字）
Who/What/When/Where/Why + 1核心数据 + 1案例

【Section 2：背后原因】（400-600字）
权威声音引用 + 逻辑推演

【Section 3：影响分析】（400-600字）
对谁有影响 + 普通人行动建议（≤3条）

【Section 4：未来展望】（300-400字）
趋势判断 + 格局提升 + 比喻/金句收尾

【结尾】（100-150字）
最强记忆点 + 转发钩子
```

### B. 数据驱动模板

```
【开篇】（100字内）
直接抛出核心数据

【数据解读1】（200字）
数据1 → 含义 → 读者利益

【数据解读2】（200字）
数据2 → 含义 → 读者利益

【案例支撑】（400字）
2-3个真实案例强化数据可信度

【行动建议】（200字）
3条以内具体操作建议

【结尾】
金句/比喻收尾
```
