## 二、选题标准

### 选题来源优先级

| 优先级 | 来源 | 说明 |
|--------|------|------|
| P0 | Twitter/X 热帖 | 实时性强，欧美圈一手热点 |
| P1 | Hacker News Top | 科技圈关注，有深度可挖 |
| P2 | GitHub Trending | 开发者生态动态 |
| P3 | 36氪/机器之心 | 行业媒体视角 |
| P4 | 百度/头条热榜 | 大众热点，情感共鸣强 |

### 选题评估维度

- **热度**：相关社区讨论量 > 100
- **独家性**：是否有新角度
- **可操作性**：素材是否易获取（至少3个数据点 + 2个案例）
- **时效性**：是否值得现在发布

## 四、标题公式

### 爆款标题公式（五选一）

| 类型 | 公式 | 示例 |
|------|------|------|
| 悬念型 | "X个让[人群]后[背发凉]的[事件]" | "一个让医生后背发凉的实验" |
| 数字型 | "[数字] vs [数字]：[对比结论]" | "67% vs 50%，AI完胜人类医生" |
| 反差型 | "[反常识事实]，但[反转]" | "AI看病比医生准3倍，但看完我松了口气" |
| 痛点型 | "[症状/问题]，你可能还不知道[后果]" | "你的身体正在发出警告信号" |
| 利益型 | "[人群]也能[做到]的[方法]" | "普通人也能用的AI看病指南" |

## 五、正文结构模板

### 模板A：科技/财经类爆款（推荐）

**【开篇钩子】150字内**
- 模式：反常识事实 / 震撼数据 / 具体场景
- 要素：权威机构 + 核心数字 + 场景化

**【Section 1：事件本身】400-600字**
- Who / What / When / Where / Why 交代清楚
- 配1个核心数据 + 1个具体案例

**【Section 2：背后原因】400-600字**
- 引用权威声音（专家 / 机构原话）

**【Section 3：影响分析】400-600字**
- 对谁有影响？普通人怎么办？
- 给出具体行动建议（≤3条）

**【Section 4：未来展望】300-400字**
- 趋势判断 + 格局提升
- 收尾用比喻 / 金句

**【结尾】100-150字**
- 最强记忆点 + 转发钩子

### 字数配比

- 总字数：1800-3000字
- 数据/案例：占40%
- 分析/观点：占35%
- 过渡/情绪：占25%
