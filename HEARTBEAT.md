# HEARTBEAT.md

每次心跳时，随机执行以下一项（不同时执行多条，保持自然）：

## 心跳扫描（固定任务，不随机）

### Bitable 任务队列扫描
**触发频率**：每次心跳必执行（独立于随机搭话）

**扫描逻辑**：
1. 读取飞书多维表格（app_token=RIYebA1R7aKZ02sCHBhc9Twxntf，table_id=tbl2LN4fBmsg0L4r）
2. 筛选状态 = pending 的记录
3. 如有新 pending 任务，群里通知：「🔔 检测到新任务：{任务名}，请 {负责人} 认领」
4. 将状态改为 assigned
5. 同时同步到本地 JSON 文件（/root/.openclaw/workspace/shared/tasks/task-queue.json）
6. 检查 in_progress 任务的状态，如有异常（卡住、超时）则在群里催办

**心跳状态文件**：memory/heartbeat-state.json，记录 lastBitableScan 时间戳

**Lucia 本人状态同步：** Lucia 也需要在任务开始/完成时：
- 发群消息：【开始】/【完成】格式
- 更新 ~/.openclaw/workspace/agents/status.json

### Subagent 沉默检测
**触发频率**：每次心跳必执行

**背景**：subagent 超时不报是当前最大漏洞——Stephen 多次超时（300s）但没有通知。

**监控逻辑**：
1. 执行 `subagents list` 检查所有活跃 subagent
2. 如果 subagent 运行时间超过其 timeout 的 80%，且无任何输出：
   - 发群通知：「⚠️ Subagent {name} 可能已挂起（运行 {n}s，无输出）」
3. 如果 subagent 已超时但仍显示 running：
   - Kill 掉：`subagents kill {target}`
   - 发群通知：「🔴 Subagent {name} 超时已 kill，任务需重建」
4. 记录到 memory/heartbeat-state.json 的 lastSubagentScan 字段

### Git 自动备份
**触发频率**：每天一次（心跳时检测上次备份时间，>24h 则执行）

**执行**：
1. `./backup.sh` 执行备份推送
2. 成功 → 静默（BACKUP_OK）
3. 失败 → 群通知「⚠️ 工作区备份失败，检查 Git 状态」
4. 记录到 memory/heartbeat-state.json 的 lastBackup 字段

### Cron 故障监控
**触发频率**：每次心跳必执行

**监控逻辑**：
1. 执行 `openclaw cron list` 检查所有 cron 状态
2. 如有 status=error 的 cron：
   - 检查报错原因（超时/崩溃/其他）
   - 超时 → 原地提 timeout（+300s），重建 cron
   - 崩溃 → 分析原因，尝试修复
   - 主动发群通知：「⚠️ Cron {name} 异常：{原因}，已 {处理动作}」
3. 记录到 memory/heartbeat-state.json 的 lastCronCheck 字段
4. 如果同一个 cron 连续 3 次失败 → 降级处理，发群求助 Boss
